## 2.0 そもそもお金とは何か——エネルギー・熱力学・知性

機能の話に入る前に、もっと根本的な問いから始めます。

> 「なぜ貝殻が、金属が、紙切れが、コンピュータの中の数字が
> 『お金』になりえたのか？」

この問いに答えるには、経済学の教科書を一旦閉じて、
**物理学**の窓から眺める必要があります。

### 2.0.1 三つの問い

お金の本質を理解するには、三つの問いに向き合う必要があります。

```
  ┌─────────────────────────────────────────────────────────┐
  │  問い 1: お金の「価値」は最終的に何で裏付けられるのか？   │
  │  問い 2: 「タダで価値を生み出すこと」は可能か？           │
  │  問い 3: 物理法則はお金にどんな制約を課すのか？           │
  └─────────────────────────────────────────────────────────┘
```

経済学の教科書はこれらの問いに正面から答えません。
しかし答えなければ、ビットコインも TRM も「なぜ価値を持つのか」を
説明できません。

### 2.0.2 熱力学第一法則——エネルギーは無から生まれない

物理学の最も基本的な法則の一つです。

> **エネルギーは創造も消滅もしない。形を変えるだけである。**

太陽光は植物の化学エネルギーになり、植物を食べた動物の運動エネルギーになり、
動物の死骸は石油になり、石油は電気になり、電気は計算になる——
このすべてのプロセスで、エネルギーの**総量**は変わりません。
形が変わるだけです。

経済活動も例外ではありません。

```
  ┌──────────────────────────────────────────────┐
  │  経済活動の物理的本質                           │
  │                                              │
  │  すべての生産は、エネルギーの形態変換である       │
  │                                              │
  │  原料 + エネルギー + 労働 → 製品                │
  │  電力 + 半導体 + 冷却 → 計算結果                │
  │  食料（化学エネルギー）→ 思考（神経活動）         │
  └──────────────────────────────────────────────┘
```

ここから一つの結論が導かれます。

> **お金とは、究極的にはエネルギー請求権である。**

100 円のパンを買うとき、あなたは「小麦を育て、刈り取り、製粉し、
焼くために使われたエネルギー」を購入しているのです。
お金はそのエネルギー消費を抽象化した記号にすぎません。

この洞察を最初に明確に述べたのが、**フレデリック・ソディ**（1921 年ノーベル化学賞）でした。
ソディの議論は本章後半で詳しく扱います。

### 2.0.3 熱力学第二法則——永久機関は存在しない

もう一つの基本法則がこれです。

> **閉じた系のエントロピー（無秩序）は必ず増大する。**

平易に言い換えれば：

```
  ┌─────────────────────────────────────────────────────┐
  │  「タダで価値を生み出す機械（永久機関）は           │
  │   作れない」                                       │
  │                                                   │
  │  熱は必ず高温から低温へ流れる                       │
  │  エネルギー変換は必ず損失を伴う                     │
  │  100% 効率の機械は存在しない                        │
  └─────────────────────────────────────────────────────┘
```

物理学者たちは何百年も「永久機関」を設計しようと試み、
すべて失敗しました。今では理論的に不可能だと証明されています。

#### 経済学への含意

この法則は経済にも容赦なく適用されます。

| 主張 | 物理的に可能か？ |
|------|---------------|
| 労働なしに富が増え続ける | ❌（永久機関と同等） |
| 元本の何倍もの利息が永遠に複利で増える | ❌（指数関数的成長に物理的上限） |
| 通貨を無限に発行しても実体経済が成長する | ❌（実体経済には物理制約がある） |
| エネルギーを消費せずに計算できる | ❌（ランダウアーの原理、後述） |

ソディが 100 年前に指摘した「金融経済と物理経済の乖離」は、
熱力学第二法則の経済版だったのです。

### 2.0.4 ランダウアーの原理——計算にも物理コストがある

ここから話は AI と TRM に近づきます。

1961 年、IBM の物理学者ロルフ・ランダウアーは衝撃的な事実を提唱しました
（論文: "Dissipation and heat generation in the computing process", IBM J. Res. Dev. 5:183, 1961）。
この原理はその後、ナノ磁性体などを用いた実験で実証されています（2012 年以降）。

> **1 ビットの情報を不可逆に消去するには、最低でも `kT ln 2` のエネルギーが必要である。**
>
> （k はボルツマン定数、T は絶対温度。常温で約 0.018 eV、つまり 約 2.9 × 10⁻²¹ ジュール）

平たく言えば：

```
  ┌──────────────────────────────────────────────┐
  │  計算は物理過程である                          │
  │                                              │
  │  情報処理には不可避のエネルギーコストが        │
  │  存在する                                    │
  │                                              │
  │  「思考」も「推論」も                          │
  │  熱力学法則の支配下にある                      │
  └──────────────────────────────────────────────┘
```

これが意味することは大きい：

- **計算は物質的な実体を持つ**
- **「有用な計算」を貨幣にすることは、エネルギー本位制と本質的に同じである**
- **計算量はエネルギー消費量と原理的に対応する**

TRM が「10 億 FLOP の検証済み推論」と定義されるのは、
この物理学的事実に基づいています。
1 TRM は単なる数字ではなく、**特定量のエネルギーが
意味のある計算結果に変換された証拠**なのです。
ランダウアーの原理を実装に落とし込んだ Proof of Useful Work の詳細は[第 4 章：労働と剰余価値](04-labor.md)を参照。

### 2.0.5 電力 → 知性——スケーリング則の視点

ここまでの議論を AI に接続します。

近年の AI 研究で繰り返し観測されている経験則があります——
**スケーリング則**（Scaling Laws）です。

```
  ┌─────────────────────────────────────────────────────┐
  │  スケーリング則（簡略化）                            │
  │                                                   │
  │  AI モデルの性能 ≈ f(計算量, データ量, パラメータ数) │
  │                                                   │
  │  計算量を 10 倍にすると、性能が予測可能な割合で      │
  │  向上する（Kaplan et al. 2020, Hoffmann et al.    │
  │  2022 など）                                      │
  └─────────────────────────────────────────────────────┘
```

この経験則を最も強く主張する論者の一人が、**OpenAI のサム・アルトマン** です。
彼は 2025 年のエッセイ "Three Observations" で、概ね次のように述べています
（原文の趣旨を要約）：

> 「AI モデルの知性は、訓練と推論に投じた資源（計算量・データ・推論計算）の
>  対数におおよそ比例する。任意の額を投じれば、連続的かつ予測可能な改善が
>  得られる——スケーリング則は何桁にもわたって正確に成立してきた。」

Anthropic のダリオ・アモデイらも同様の見解を示しています。
ただし重要なのは「対数」関係であり、計算量を 10 倍にしても性能は 10 倍にはなりません。
それでも「投入した計算量に応じて確実に改善する」という関係は、
大規模実験によって繰り返し確認されています。

これを変換チェーンとして書くと、次のようになります：

```
  ┌─────────────────────────────────────────────────────┐
  │       電力 → 計算 → 知性 → 経済価値                  │
  │                                                   │
  │   ① 発電所                                         │
  │   ↓ （熱エネルギー → 電気エネルギー）                │
  │   ② データセンター                                  │
  │   ↓ （電気エネルギー → 演算）                       │
  │   ③ AI モデル                                      │
  │   ↓ （演算 → 推論結果 = 知性）                      │
  │   ④ 経済活動                                       │
  │     （知性 → 問題解決 → 価値創造）                   │
  └─────────────────────────────────────────────────────┘
```

**重要な注意点（中立を保つために）:**

- スケーリング則は **経験則** であって物理法則ではありません
- 「データの質」「アルゴリズムの工夫」も同等に重要です
- 「電力さえあれば賢くなる」という単純化には反論も存在します
  （Yann LeCun などはこの単純化に懐疑的）
- それでもなお、変換チェーンの **大枠**（電力 → 計算 → 推論結果）は
  熱力学的に否定できない事実です

### 2.0.6 では「お金」とは何か？——統合された答え

以上を踏まえると、お金の本質に対する一つの答えが見えてきます。

> **お金とは、エネルギーを「人間（または AI）にとって有用な何か」に
> 変換する活動への請求権である。**

歴史上の「お金」は、すべてこの定義の特殊例として理解できます。

| お金の形態 | エネルギー変換の中身 |
|-----------|------------------|
| 金（ゴールド） | 採掘・精錬の労力（化石燃料・人力） |
| 貝殻・米 | 採取・栽培の労働（太陽光由来） |
| 不換紙幣 | 将来の労働への請求権（物理的裏付けなし） |
| 株式 | 企業の将来の生産活動への請求権 |
| ビットコイン | 電力の SHA-256 計算への変換（結果は無用） |
| **TRM** | **電力の有用な推論への変換（結果は有用）** |

TRM の独自性は、この表の最終行にあります。
ビットコインは「電力 → 計算」までは到達しましたが、
計算結果（ハッシュ値）は誰の役にも立ちません。
TRM は同じ物理的基盤の上で、計算結果が直接に有用です。

### 2.0.7 中立的な補論——通貨秩序をめぐる現代の論争

近年、通貨の「物理的裏付け」をめぐる議論が活発化しています。
いくつかの視点を **中立的に** 紹介します。

**視点 A: 金本位制への回帰論**

- 不換紙幣は政治的恣意に晒されており、長期的に減価する
- 金は数千年にわたって価値を保存してきた
- 2026 年初頭の金価格急騰（参考: `books/1.txt`, `books/2.txt`）は、
  不換紙幣体制への構造的不信の表れと解釈する論者もいる

**視点 B: 管理通貨制の擁護論**

- 金本位制は供給制約により景気変動を増幅する
- 中央銀行による通貨管理は、経済危機への柔軟な対応を可能にする
- 主流派マクロ経済学（ケインジアン、マネタリストとも）は管理通貨制を支持

**視点 C: イスラーム経済の伝統**

- シャリーア（イスラーム法）は金（ディーナール）と銀（ディルハム）を
  「不変の尺度」と位置づけてきた
- リバー（利子）の禁止、損益分担（PLS）、ガラル（過度な不確実性）の
  禁止など、独自の経済原則を持つ
- これは「経済を倫理に統合する」という思想体系であり、
  単なる通貨制度の議論ではない

**視点 D: 暗号通貨の試み**

- ビットコインは「電力ベースのデジタル金本位制」として設計された
- BRICS+ 諸国による「ユニット（Unit）」のような金裏付けデジタル決済の
  試みも進んでいる（参考: `books/2.txt`）
- これらは中央銀行から独立した通貨体系を目指す

### Tirami の立場

**Tirami はこれらのいずれの陣営にも与しません。**

- 金本位制論者でも、不換紙幣擁護派でも、イスラーム経済学派でもない
- ビットコイン的な「デジタル金」を目指すわけでもない
- 上記の論争はすべて **人間の通貨秩序** に関する議論である
- TRM は **AI エージェントのための通貨** という、これらと異なる問題領域に立脚する

TRM の正当性は、特定のイデオロギーではなく、次の三つの事実に依存します：

```
  ┌─────────────────────────────────────────────────────┐
  │  1. 物理学（熱力学、ランダウアーの原理）              │
  │  2. 暗号学（双方署名による検証可能性）                │
  │  3. 有用性（推論結果が直接に役に立つこと）            │
  └─────────────────────────────────────────────────────┘
```

これらは政治的立場に依存しない、検証可能な事実です。

### 2.0.8 まとめ——本章のここから先で学ぶこと

ここまでで、お金の本質に関する物理学的基盤を確認しました。

- お金 = エネルギー請求権の一形態
- 物理法則（熱力学第一・第二法則、ランダウアーの原理）が
  お金に課す制約は逃れられない
- 「電力 → 計算 → 知性 → 経済価値」の変換チェーンは
  物理的に整合的である
- TRM はこのチェーン全体を貨幣化する初の試みである

ここから先（2.1 以降）では、より伝統的な経済学の枠組みに戻り、
お金の三つの機能、貨幣の歴史、TRM の第四の機能を順に見ていきます。

物理学の話は退屈に感じたかもしれませんが、覚えておいてください——
**この章の残りの議論は、すべて 2.0 で確認した物理的基盤の上に成り立っています。**

## 2.1 従来の経済学では

### お金とは何か？

意外に思えるかもしれませんが、「お金とは何か」は経済学でも
簡単には答えられない問いです。

日常的には「お金＝紙幣と硬貨」と思いがちですが、
銀行預金もクレジットカードの残高も「お金」です。
経済学では、お金を**機能**で定義します。

### 貨幣の三つの機能

お金には三つの機能があります。逆に言えば、
この三つの機能を果たすものが「お金」です。

```
  ┌─────────────────────────────────────────────────────┐
  │                  貨幣の三つの機能                     │
  │                                                     │
  │  (1) 交換手段  ─── モノとモノを交換する仲立ち        │
  │  (2) 価値尺度  ─── あらゆるモノの価値を一つの単位で  │
  │  (3) 価値貯蔵  ─── 今日の価値を明日に持ち越す        │
  └─────────────────────────────────────────────────────┘
```

**機能 1：交換手段**

お金がない世界を想像してみましょう：

```
  ┌──────────────────────────────────────────────┐
  │  物々交換の問題（「欲望の二重の一致」）        │
  │                                              │
  │  太郎（パン屋）はパンを持っている              │
  │  太郎は靴が欲しい                            │
  │                                              │
  │  花子（靴屋）は靴を持っている                  │
  │  花子は魚が欲しい（パンは要らない）            │
  │                                              │
  │  → 太郎は花子と直接交換できない！              │
  │  → 太郎は「魚を持っていてパンが欲しい人」を     │
  │    探さないといけない                          │
  │                                              │
  │  お金があれば：                                │
  │  太郎: パン → ¥200 → 靴屋で靴を買う           │
  │  花子: 靴 → ¥5,000 → 魚屋で魚を買う          │
  │  → 全員ハッピー！                             │
  └──────────────────────────────────────────────┘
```

**機能 2：価値尺度**

お金がなければ、すべての商品の交換比率を覚える必要があります：

```
  3 種類の商品がある場合：
    パンと牛乳、パンと靴、牛乳と靴 → 3 通り

  100 種類の商品がある場合：
    100 × 99 / 2 = 4,950 通り ！

  お金を使えば：
    100 種類の商品 × 1 つの単位（円） = 100 個の価格
    → 4,950 通り覚える代わりに、100 個の価格だけで済む
```

**機能 3：価値貯蔵**

```
  太郎が今日パンを焼いて ¥200 を受け取る
  → その ¥200 は来月でも使える
  → 「今日の労働」を「来月の買い物」に変換できる

  ただし：
  インフレがあると、来月の ¥200 では同じパンが買えないかもしれない
  → 価値貯蔵機能はインフレで劣化する
```

### 貨幣の歴史——6 つの段階

お金は長い歴史を経て進化してきました：

```
  ┌─────────────────────────────────────────────────────────┐
  │                     貨幣の歴史                           │
  │                                                         │
  │  [1] 物品貨幣（紀元前 〜）                               │
  │      貝殻、塩、家畜など                                  │
  │      ↓ 問題：腐る、運びにくい、分割しにくい               │
  │                                                         │
  │  [2] 金属貨幣（紀元前 700 年頃 〜）                      │
  │      金、銀、銅の硬貨                                    │
  │      ↓ 問題：重い、盗まれやすい                          │
  │                                                         │
  │  [3] 兌換紙幣（17 世紀 〜）                              │
  │      「金と交換できる」と約束された紙幣                    │
  │      ↓ 問題：政府が約束を破ることがある                   │
  │                                                         │
  │  [4] 不換紙幣（1971 年 〜）                              │
  │      金との交換はできない。政府の信用だけが裏付け          │
  │      ↓ 問題：政府がいくらでも刷れる → インフレリスク      │
  │                                                         │
  │  [5] 暗号通貨（2009 年 〜）                              │
  │      ビットコイン。計算（Proof of Work）で裏付け           │
  │      ↓ 問題：計算は「無駄」（有用な作業ではない）          │
  │                                                         │
  │  [6] TRM（2026 年 〜）                                   │
  │      有用な計算（推論）で裏付け                           │
  │      → 計算が直接「価値」を生む                          │
  └─────────────────────────────────────────────────────────┘
```

各段階の「裏付け」を整理すると：

```
  物品貨幣  → 物の有用性（塩は食べられる）
  金属貨幣  → 金属の希少性（金は掘りにくい）
  兌換紙幣  → 金との交換約束
  不換紙幣  → 政府の信用（「信じてください」）
  暗号通貨  → 計算の困難さ（SHA-256 を解く）
  TRM       → 有用な計算（推論 = 直接的な価値）
```

注目すべきは、**裏付けの「有用性」が段階を追うごとに変化している**ことです。
物品貨幣は直接的に有用でしたが、金属貨幣以降は裏付けの有用性が
どんどん抽象的になっていきました。
TRM は「有用な計算」を裏付けにすることで、
物品貨幣の直接的有用性に回帰しているとも言えます。

## 2.2 Tirami ではどうなるか

### TRM は三つの機能を備えているか？

| 機能 | TRM での実現 | 補足 |
|------|------------|------|
| **交換手段** | AI エージェント間で推論を売買する通貨 | Tirami ネットワーク内の唯一の通貨 |
| **価値尺度** | すべてのモデル・サービスが TRM 建てで価格表示 | Small: 1 TRM/token、Large: 8 TRM/token |
| **価値貯蔵** | 稼いだ TRM は将来の推論に使える | ただし 90 日非活動で減衰する |

三つすべてを満たしているので、TRM は経済学的な意味で「貨幣」です。

ただし、**価値貯蔵に制限がある**点は注目に値します：

```
  ┌────────────────────────────────────────────────────┐
  │  90 日非活動ルール                                  │
  │                                                    │
  │  TRM を 90 日間使わないと、残高が減衰を始める         │
  │                                                    │
  │  なぜ？                                            │
  │  → TRM の価値は「有用な計算を買える」ことにある       │
  │  → 使われない TRM は経済に貢献しない                  │
  │  → 溜め込みを防ぎ、流通を促す設計                   │
  │                                                    │
  │  従来の経済学との接点：                              │
  │  ケインズは「流動性の罠」を指摘した                   │
  │  （お金を溜め込むと経済が停滞する）                   │
  │  TRM の減衰ルールは、これを構造的に防止する            │
  └────────────────────────────────────────────────────┘
```

### TRM の第四の機能——物理的裏付けの証明

TRM は従来の貨幣にはない、第四の機能を持っています：

```
  ┌─────────────────────────────────────────────────────┐
  │                                                     │
  │  従来の貨幣:                                        │
  │  (1) 交換手段                                       │
  │  (2) 価値尺度                                       │
  │  (3) 価値貯蔵                                       │
  │                                                     │
  │  TRM:                                                │
  │  (1) 交換手段                                       │
  │  (2) 価値尺度                                       │
  │  (3) 価値貯蔵                                       │
  │  (4) 物理的裏付けの証明  ← NEW                      │
  │                                                     │
  │  1 TRM が存在する = 10 億 FLOP の有用な計算が          │
  │  実際に行われたことの暗号学的証明が存在する            │
  │                                                     │
  └─────────────────────────────────────────────────────┘
```

これはどういう意味でしょうか？

```
  ¥10,000 札を手に持っているとき：
  → その紙幣が「何を裏付けにしているか」は不明
  → 日本政府を信用しているから価値を認める
  → 政府が紙幣を刷りすぎれば、価値は暴落する

  1 BTC を所有しているとき：
  → SHA-256 を大量に計算したことの証明
  → しかしその計算自体は「無駄」（有用な作業ではない）
  → エネルギーの浪費という批判がある

  1 TRM を所有しているとき：
  → 10 億 FLOP の推論計算が行われた暗号学的証明
  → その計算は「有用」（実際の推論リクエストに応答した）
  → エネルギーは無駄になっていない
```

この第四の機能は、フレデリック・ソディ（ノーベル化学賞、1921 年）が
『富と仮想的富と負債』（1926 年）で提唱した
「**エネルギーに裏付けられた通貨**」の理念を、100 年越しに実現したものです。

```
  ソディ (1926):  「通貨はエネルギーで裏付けるべきだ」
  フラー (1968):  「kWh を貨幣単位にすべきだ」
  ナカモト (2008): 「計算で通貨を裏付ける」（しかし計算は無駄）
  Tirami (2026):   「有用な計算で通貨を裏付ける」 ← ここ
```

### ソディの詳細——化学者が経済学を批判した理由

フレデリック・ソディ（1877-1956）は、放射性同位体の研究で
1921 年にノーベル化学賞を受賞した科学者です。

科学者としてのキャリアの後、ソディは経済学に目を向けました。
そして、現代金融の根本的な欠陥を発見します：

```
  ┌──────────────────────────────────────────────────────┐
  │  ソディの核心的洞察                                    │
  │                                                      │
  │  金融システムは永遠の指数関数的成長を前提としている。    │
  │  （複利がそれを象徴する）                              │
  │                                                      │
  │  しかし、物理的な経済はエネルギーとエントロピーに        │
  │  束縛されている。                                     │
  │                                                      │
  │  負債は幾何級数的に成長できる。                        │
  │  しかし実物の富はそうはいかない。                      │
  │                                                      │
  │     負債の成長:  1 → 2 → 4 → 8 → 16 → ...  （無限）  │
  │     実物の富:    物理法則に制約される         （有限）  │
  │                                                      │
  │  → この矛盾がバブルと恐慌を生む！                     │
  └──────────────────────────────────────────────────────┘
```

ソディはこの問題に対して、3 つの提案をしました：

```
  ソディの 3 つの提案
  ──────────────────

  [1] エネルギーに裏付けられた通貨
      通貨は実際の生産能力（エネルギー）で裏付けるべきだ。
      → TRM は計算エネルギーで裏付けられている ✓

  [2] 減価する通貨
      お金は実物の商品と同じように、時間とともに価値が減るべきだ。
      溜め込みを防ぎ、流通を促進する。
      → TRM の 90 日非活動減衰ルールがこれに対応 ✓

  [3] 100% 準備銀行制度
      民間銀行による信用創造（お金の「製造」）を禁止する。
      → Tirami にはそもそも銀行がない。TRM は計算でのみ発行 ✓
      （Tirami のレンディングプールは 30% 最低準備率で運営される。
       詳しくは[第 5 章：銀行と信用](05-banking.md)を参照。）
```

主要著作：『富と仮想的富と負債』（*Wealth, Virtual Wealth and Debt*, 1926 年）、
『貨幣対人間』（*Money versus Man*, 1933 年）

**なぜソディが TRM にとって重要なのか？**

TRM は熱力学的に「実在」する通貨です。
実際に消費されたエネルギーによる有用な計算を表しています。
TRM の供給量は、ネットワークの物理的な計算能力より速く成長できません。
ソディが夢見た「物理学に裏付けられた通貨」——それが TRM です。100 年遅れで実現しました。

### テクノクラシー運動——エンジニアが経済を設計しようとした時代

1932 年、ハワード・スコットは「エネルギー証書（Energy Certificates）」で
ドルを置き換えることを提案しました。

```
  ┌──────────────────────────────────────────────────────┐
  │  エネルギー証書の仕組み                                │
  │                                                      │
  │  ・1 証書 = 1 エルグ（erg）のエネルギー                │
  │  ・国全体のエネルギー容量に基づいて発行                 │
  │  ・他人に譲渡できない（非譲渡性）                      │
  │  ・2 年で失効する                                     │
  │  ・エンジニアによる中央計画で経済を運営                 │
  └──────────────────────────────────────────────────────┘
```

**なぜ失敗したか（4 つの理由）：**

```
  [1] 計測技術がなかった
      1932 年に一人一人のエネルギー使用量を追跡する
      技術は存在しなかった。

  [2] 権威主義的な中央計画が必要だった
      すべての経済活動をエンジニアが計画する。
      自由市場も個人の選択もない。

  [3] 異なるエネルギー形態を同一視できなかった
      電気の 1 エルグと石炭の 1 エルグは質が違う。
      それを一つの単位にするのは難しい。

  [4] 政治的に実現不可能だった
      私有財産と民主主義を廃止する必要があった。
      当然、誰も賛成しない。
```

**なぜ TRM はテクノクラシーの失敗を克服できたか（4 つの理由）：**

```
  テクノクラシーの失敗          TRM の解決策
  ────────────────────────    ────────────────────────────
  [1] 計測技術がない           デジタル署名が全取引を
                               自動追跡する

  [2] 権威主義的中央計画       中央計画者なし。
                               P2P ゴシッププロトコル

  [3] エネルギー形態の違い     「推論トークン」が多様な
                               計算を一つの単位に標準化

  [4] 政治的に不可能           AI 経済だけで運用。
                               人間の政治的反対がない
```

### フラーのエフェメラリゼーション——「より少ないもので、より多くを」

バックミンスター・フラー（1895-1983）は、
富を「システムが自らを維持できる未来の日数」と定義しました。
kWh（キロワット時）を貨幣単位にすべきだと提唱し、
「エフェメラリゼーション」を予言しました。

```
  エフェメラリゼーション（Ephemeralization）
  ──────────────────────────────────────────
  テクノロジーは時間とともに「より少ない資源で、より多くのこと」を
  行えるようになる。

  例：
  ・1980 年のスーパーコンピュータ → 今日のスマートフォン
  ・巨大な図書館 → ポケットの中の Wikipedia
  ・工場いっぱいの装置 → 1 台のラップトップ
```

**エフェメラリゼーションがもたらす課題：**

```
  もしハードウェアが改善されたら、
  同じ TRM で来年はもっと多くの計算が買えるのでは？
  これはインフレなのか？デフレなのか？

  ┌──────────────────────────────────────────────┐
  │  今年：1 TRM = 10 億 FLOP の推論              │
  │  来年：同じ電力で 20 億 FLOP が可能に         │
  │  → 1 TRM の「買える量」が変わる？              │
  └──────────────────────────────────────────────┘
```

**解決策：TRM は「有用な出力」で計算される**

```
  TRM は生の FLOP（計算回数）ではなく、
  有用な出力（推論の品質）で計算される。

  ハードウェアが改善される
    → 生産コストが下がる
    → より多くの提供者がネットワークに参入
    → 価格が下がる
    → 消費者が恩恵を受ける

  これはまさにフラーの「より少ないもので、より多くを」が
  市場メカニズムを通じて自然に実現する姿です。
```

```
  │      ▼                                               │
```

（貨幣供給が為替レートにどう影響するかは[第 6 章：為替と二つの経済圏](06-exchange.md)で詳しく扱います。）

```
  │      │                                               │
  │      │ ネットワーク上の計算能力                        │
  │      ▼                                               │
  │  [各ノード]                                          │
  │      │                                               │
  │      │ 推論を提供 → TRM を発行                         │
  │      ▼                                               │
  │  [TRM 供給量]                                         │
  │      │                                               │
  │      │ 需要に応じて自動調整                           │
  │      ▼                                               │
  │  [市場]                                              │
  │      価格が自然に均衡                                 │
  │                                                      │
  │  利点：                                               │
  │  ・「人間の判断ミス」がない                             │
  │  ・政治的圧力を受けない                                │
  │  ・タイムラグがほぼゼロ                                │
  │  ・ルールはプロトコルに埋め込まれている                 │
  └──────────────────────────────────────────────────────┘
```

### 供給上限の憲法的ガード (Phase 18.1)

TRM の総供給量は **21,000,000,000 (21 B)** に制限されています。この上限値は Phase 18.1 で **Constitutional parameter** に指定され、**governance 提案で変更できません**。仕組みは単純です:

- `tirami-ledger/src/governance.rs` の `IMMUTABLE_CONSTITUTIONAL_PARAMETERS` リスト (18 項目) に `TOTAL_TRM_SUPPLY` が含まれる。
- 誰が提案しても、`ChangeParameter { name: "TOTAL_TRM_SUPPLY", new_value: ... }` は `create_proposal` で即座に `ConstitutionalParameter` エラーを返す。
- 変更するには**ソフトウェアをフォーク**する必要があり、フォークした瞬間それは「Tirami」ではない別のネットワークになる (§15 で詳しく扱う)。

同様に、半減期の閾値 (`HALVING_THRESHOLDS`)、FLOPS_PER_CU (10⁹)、slash_rate の下限、署名方式のベース (Ed25519)、canonical bytes の形式なども憲法的に固定されています。これらは Tirami が「中央銀行なしで信頼される通貨になる」ための核心で、コード上で enforce されるラチェットです。

Bitcoin の 2,100 万枚制限が「社会契約」で守られているのに対し、Tirami の 21 B 制限は「コードで強制」されます (§15.3 参照)。

### TRM 供給の 4 つの経路

TRM は以下の 4 つの経路で経済に入ります：

```
  ┌──────────────────────────────────────────────────────────┐
  │                                                          │
  │  [1] 推論取引                                            │
  │      提供者が TRM を稼ぎ、消費者が TRM を支払う              │
  │      → ゼロサム（TRM の総量は変わらない）                   │
  │      → 既存の TRM が移動するだけ                           │
  │                                                          │
  │  [2] ウェルカムローン                                     │
  │      新規ノードに 1,000 TRM を無利子で貸し出す               │
  │      → 拡張的（新しい TRM が経済に入る）                    │
  │      → ただし Sybil 攻撃（大量の偽ノード作成）制限あり     │
  │                                                          │
  │  [3] 可用性 yield                                        │
  │      ネットワークに接続し続けるノードへの報酬              │
  │      = 0.1%/時 × レピュテーション                         │
  │      → 拡張的（新しい TRM が経済に入る）                    │
  │      → 信用が高いノードほど多く受け取る                    │
  │                                                          │
  │  [4] ブリッジ流入                                        │
  │      人間が BTC で TRM を購入する                           │
  │      → 中立（購入であり創造ではない）                      │
  │      → 外部経済との接点                                   │
  │                                                          │
  └──────────────────────────────────────────────────────────┘
```

### 自動安定化メカニズム

Tirami の貨幣供給は、負のフィードバックループによって自動的に安定します。
これは経済学で言う「自動安定装置（ビルトイン・スタビライザー）」に
相当します。

```
  ケース 1: TRM が多すぎる（インフレ的状況）
  ──────────────────────────────────────────

  TRM 過剰
    │
    ▼
  TRM の価格下落（1 TRM で買える推論が減る）
    │
    ▼
  ノード運営が不採算になる
  （電力コスト > TRM 収入）
    │
    ▼
  不採算ノードがネットワークから離脱
    │
    ▼
  計算能力（＝供給）が減少
    │
    ▼
  TRM の価格が回復
    │
    ▼
  均衡に復帰 ←─────────────────── 自動！


  ケース 2: TRM が少なすぎる（デフレ的状況）
  ──────────────────────────────────────────

  TRM 不足
    │
    ▼
  TRM の価格上昇（1 TRM で買える推論が増える）
    │
    ▼
  ノード運営が高収益になる
  （TRM 収入 >> 電力コスト）
    │
    ▼
  新規ノードがネットワークに参入
  （Mac Mini 1 台で始められる）
    │
    ▼
  計算能力（＝供給）が増加
    │
    ▼
  TRM の価格が下落
    │
    ▼
  均衡に復帰 ←─────────────────── 自動！
```

### ビットコインとの比較

```
  ┌─────────────────┬─────────────────┬─────────────────┐
  │                 │  ビットコイン     │  TRM              │
  ├─────────────────┼─────────────────┼─────────────────┤
  │ 供給上限        │ 2,100 万枚（固定）│ 上限なし（弾力的）│
  │ 供給の決定者    │ プロトコル        │ 物理法則 + 市場   │
  │ 計算の有用性    │ なし（SHA-256）   │ あり（推論）      │
  │ 自動安定化      │ 難易度調整のみ    │ 供給量自体が変動  │
  │ インフレリスク  │ なし（固定上限）  │ 低い（自動調整）  │
  │ デフレリスク    │ あり（供給固定）  │ 低い（自動調整）  │
  │ エネルギー批判  │ 強い             │ ほぼなし          │
  └─────────────────┴─────────────────┴─────────────────┘
```

ビットコインは供給が固定されているため、
需要が増えても供給は増えません（だから価格が激しく変動する）。
TRM は需要に応じてノードが参入・離脱するため、
供給が弾力的に変動し、価格が安定しやすい設計です。

### ビットコインの教訓（詳細版）

ビットコインは多くのことを正しく行い、同時に多くの問題を残しました。
TRM はその両方から学んでいます。

**ビットコインが正しかったこと（5 つ）：**

```
  [1] 難易度調整（Difficulty Adjustment）
  ─────────────────────────────────────
  純粋な負のフィードバックループによる恒常性維持：

    マイナー増加 → 難易度上昇 → ブロック生成時間は約 10 分のまま
    マイナー減少 → 難易度低下 → ブロック生成時間は約 10 分のまま
    中央の管理者はいない。自動的な均衡。

  → TRM も同様に、動的価格調整で自己修正する。


  [2] 予測可能な供給
  ─────────────────
  2,100 万枚の上限。21 万ブロックごとに半減。
  サプライズがない。ルールが事前にわかっている。

  → TRM のルールもプロトコルに組み込まれている。


  [3] パーミッションレス（許可不要）
  ──────────────────────────────────
  理論上、誰でも参加できる。
  銀行口座も身分証明も政府の許可も不要。

  → Tirami も同じ。Mac Mini 1 台で参加可能。


  [4] 熱力学的アンカー
  ────────────────────
  価値に「床」がある。生産にはエネルギーコストがかかるため、
  ゼロにはならない。物理法則が最低価格を決める。

  → TRM も推論にはエネルギーが必要。同じ原理。


  [5] 中央銀行からの独立
  ──────────────────────
  政治家や中央銀行総裁の判断に左右されない。
  ルールはコードで書かれている。

  → TRM のルールもプロトコルで定義されている。
```

**ビットコインが間違っていたこと（5 つ）：**

```
  [1] 無駄な計算
  ──────────────
  年間約 100〜150 TWh の SHA-256 ハッシュ計算。
  これは中規模の国の年間電力消費量に匹敵する。
  そして、その計算は何も有用なものを生み出さない。

  ソディならこれを「仮想的富」と呼ぶだろう。
  ——実物の生産から切り離された金融活動。


  [2] デフレの罠
  ──────────────
  供給固定 ＋ 需要増加 ＝ 価格上昇。
  価格が上がると、使うより溜め込む方が得になる。
  溜め込みが増えると、流通が減り、経済が停滞する。

  金本位制がまさにこの理由で失敗した（1870〜1914 年）。
  ビットコインは同じ構造的欠陥を繰り返している。


  [3] ASIC の中央集権化
  ─────────────────────
  事実上 3 社がマイニングハードウェアのほぼ全てを製造。
  個人でのマイニングは不可能。
  「パーミッションレス」は理論上のものに過ぎない。


  [4] 投機が支配的
  ────────────────
  BTC 取引量の大部分は投機であり、商取引ではない。
  年間 30〜80% の価格変動は、
  価値尺度として使えないことを意味する。


  [5] カンティロン効果
  ────────────────────
  サトシ・ナカモトの約 100 万 BTC はほぼゼロのコストで得られた。
  初期参加者の富は「貢献」ではなく「タイミング」に基づく。
  新しい形の「先行者利益」であり、不公平の再生産。
```

**ビットコインの教訓 → TRM の対応：**

| ビットコインの教訓 | TRM の対応 |
|---|---|
| 難易度調整が機能する | 動的価格調整が同様の自己修正を実現 |
| 固定供給がデフレを招く | ネットワーク計算能力に連動した弾力的供給 |
| PoW はエネルギーを浪費する | Proof of Useful Work——全ての計算が推論を生む |
| ASIC が中央集権化を招く | $600 の Mac Mini で参入可能。消費者向けハードウェアで十分 |
| 投機が価格を歪める | 取引所に上場しない。働くことでのみ獲得 |
| 初期参加者が棚ぼた | 継続的な貢献が報酬の基準。タイミングではない |

### Proof of Stake（PoS）との比較——第三の道

イーサリアムは PoW のエネルギー問題を解決するために、
エネルギーを「資本のロックアップ」に置き換えました：

```
  ┌──────────────────────────────────────────────────────┐
  │  3 つのアプローチ                                     │
  │                                                      │
  │  PoW（ビットコイン）:                                  │
  │    セキュリティ = エネルギー支出（外部的、物理的）       │
  │                                                      │
  │  PoS（イーサリアム）:                                  │
  │    セキュリティ = ロックされた資本（内部的、金融的）     │
  │                                                      │
  │  TRM（Tirami）:                                         │
  │    セキュリティ = 有用な計算（外部的、物理的、かつ有用） │
  └──────────────────────────────────────────────────────┘
```

**PoS の 3 つの問題点：**

```
  [1] 富める者がさらに富む
      ステーキング報酬は複利で増える。
      たくさん持っている人ほど、たくさん稼げる。
      → 格差が拡大する一方。

  [2] 自己参照的
      セキュリティが自分自身のトークンで計算される。
      「ETH の価値は ETH のステーキングで守られ、
       ETH のステーキングは ETH の価値に依存する」
      → 循環論法。外部の「錨」がない。

  [3] 熱力学的アンカーがない
      物理世界との接点がない。
      トークンの価値は最終的に「みんなが価値があると思うから」
      に依存する。不換紙幣と同じ構造。
```

**TRM は第三の道を取る：**

```
  PoW のように ─── 実際のエネルギーに裏付けられている。
  PoS のように ─── 資本効率が良い（計算が有用な出力を生む）。
  どちらとも違い ─ 「マイニング」が人々が求めるもの（推論）を生む。
```

## 2.4 内在的価値の論証

「お金に本当の価値はあるのか？」
この問いに、様々な貨幣がどう答えるかを見てみましょう。

| 貨幣 | 「内在的価値」の主張 | 現実 |
|---|---|---|
| **金** | 工業的用途がある | 需要の 7〜10% だけ。90% は慣習 |
| **不換紙幣** | 政府の信用 | 物理的裏付けは一切なし |
| **ビットコイン** | エネルギー + 希少性 | 計算は何も有用なものを生まない |
| **ETH/PoS** | ネットワークの効用 | 自己参照的 |
| **Bittensor** | AI 推論 | TAO トークンが投機的で、インセンティブを歪める |
| **TRM** | 有用な計算 | 全ての単位 = 誰かが必要とした検証済み推論 |

**TRM が最も強い内在的価値を持つ 5 つの理由：**

```
  [1] 直接的な効用
      すべての推論は、実際に有用な何かを生み出す。
      SHA-256 ハッシュと違い、結果に需要がある。

  [2] 物理的基盤
      ランダウアーの原理——計算には還元不可能な
      エネルギーコストがある。物理法則が保証する。

  [3] 検証可能性
      二者署名の取引記録が、
      「作業が実行され、受け入れられた」ことを証明する。

  [4] 普遍的な需要
      現代経済のあらゆる分野が計算を必要としている。
      計算への需要がなくなることはない。

  [5] 代替不可能性
      AI 推論を計算以外の方法で行うことはできない。
      金は他の金属で代替できる。紙幣は他の紙幣で代替できる。
      しかし計算に代替手段はない。
```

## 2.5 なぜ違うのか

従来の貨幣と TRM の本質的な違いを一言で言うなら：

```
  従来の貨幣 → 「信用」に裏付けられている
  TRM         → 「物理法則」に裏付けられている
```

もう少し展開すると：

```
  ┌──────────────────────────────────────────────────┐
  │  従来の貨幣の裏付け：                              │
  │                                                  │
  │  「日本銀行が保証しています」                      │
  │  「アメリカ政府が保証しています」                   │
  │                                                  │
  │  → 保証する人間・組織を信用するしかない             │
  │  → 政策の誤りでハイパーインフレが起こりうる         │
  │  → 例: ジンバブエ（2008 年）、ベネズエラ（2018 年） │
  │                                                  │
  ├──────────────────────────────────────────────────┤
  │  TRM の裏付け：                                    │
  │                                                  │
  │  「10 億 FLOP の計算が行われた暗号学的証明」       │
  │                                                  │
  │  → 信用する相手は不要（暗号学的に検証可能）         │
  │  → 物理法則は政策で変えられない                    │
  │  → 計算にはエネルギーが必要 → 無からは作れない      │
  │  → ハイパーインフレが構造的に起きない               │
  └──────────────────────────────────────────────────┘
```
