## 8.0 「市場の失敗」とは何か

ミクロ経済学の教科書では、「完全競争市場」が
最も効率的な資源配分を実現することが証明されています
（厚生経済学の第一基本定理）。

しかし現実の市場は「完全競争」からかけ離れています。
市場メカニズムだけでは効率的な結果が得られない状況を、
経済学者は **「市場の失敗（Market Failure）」** と呼びます。

```
完全競争市場の条件          現実
──────────────────────────────────
多数の売り手と買い手    →   寡占が多い（GAFA）
同質の財                →   ブランド差異
完全情報                →   売り手が品質を隠す
自由な参入・退出         →   規制、初期投資
合理的行動              →   感情的行動（バブル）
外部性がない            →   公害、騒音
```

これらの問題に対して、従来の経済学は主に
「政府の介入」で解決を図ってきました。

Tirami のアプローチは根本的に異なります。
市場の失敗の原因そのものを、プロトコル設計で除去します。
（Tirami の TRM 市場が完全競争にどれだけ近いかは[第 3 章：需要と供給](03-supply-demand.md)を参照。）

## 8.1 情報の非対称性

### 従来の経済学では——レモン市場問題

ジョージ・アカロフは1970年の論文「レモン市場」で、
情報の非対称性が市場を崩壊させるメカニズムを示しました。
（この業績により2001年にノーベル経済学賞を受賞。）

「レモン」とは、アメリカの俗語で「不良品」を意味します。

```
【中古車市場の例（レモン市場）】

  売り手: 自分の車の品質を知っている
  買い手: 外見からは品質がわからない

  ┌──────────────────────────────────────────────────┐
  │                                                  │
  │  市場に2種類の車がある:                            │
  │  ・良い車（ピーチ）: 本当の価値 100 万円            │
  │  ・悪い車（レモン）: 本当の価値 30 万円             │
  │                                                  │
  │  買い手は見分けられない → 平均的な価格 65 万円で購入  │
  │                                                  │
  │  結果:                                            │
  │  ・良い車の売り手: 65 万円では売りたくない → 撤退    │
  │  ・悪い車の売り手: 65 万円は十分 → 残る             │
  │  → 市場にレモンだけが残る（逆選択）                 │
  │  → 買い手はさらに価格を下げる                      │
  │  → 良い車がますます出てこなくなる                   │
  │  → 最終的に市場が崩壊する可能性                    │
  │                                                  │
  └──────────────────────────────────────────────────┘
```

```
従来の解決策:
  ├── 保証（メーカー保証、延長保証）
  ├── 第三者認証（車検、品質認証マーク）
  ├── 法律（消費者保護法、クーリングオフ）
  └── 評判メカニズム（口コミ、レビュー）

  問題: すべて不完全。偽レビュー、認証の不正、保証の限界
```

### Tirami ではどうなるか

```
Tirami での情報の非対称性:

  売り手（推論提供者）: 自分の推論の品質を知っている
  買い手（推論消費者）: 品質を事前に完全には知らない

  ……しかし、以下の仕組みで非対称性が大幅に軽減される
```

```
┌──────────────────────────────────────────────────────────┐
│  Tirami の情報対称化メカニズム                               │
│                                                          │
│  1. 暗号署名付き取引記録                                   │
│     ・すべての取引に提供者と消費者の Ed25519 署名            │
│     ・偽造不可能                                          │
│     ・ゴシップでネットワーク全体に伝播                      │
│                                                          │
│  2. レピュテーションシステム                                │
│     ・過去の取引履歴から自動計算                            │
│     ・各ノードが独立に検証可能                              │
│     ・偽のレピュテーションは暗号学的に不可能                 │
│                                                          │
│  3. モデルティアの明示                                     │
│     ・提供者がどのモデル（何 B パラメータ）を使っているか明示  │
│     ・「品質」の主要な決定要因が公開情報                     │
│                                                          │
│  4. リアルタイムの品質フィードバック                         │
│     ・推論結果をストリーミングで受信                        │
│     ・品質が低ければ即座に別の提供者に切り替え可能           │
└──────────────────────────────────────────────────────────┘
```

| 比較項目 | 従来の市場 | Tirami |
|---------|-----------|-------|
| 品質情報 | 売り手だけが知っている | 暗号署名付きの取引履歴で公開 |
| レビュー | 偽造可能（やらせレビュー） | 偽造不可能（暗号署名） |
| 認証機関 | 第三者が独占（利益相反の可能性） | 各ノードが独立に検証 |
| 品質基準 | 主観的（「おいしい」「きれい」） | 客観的（モデルサイズ、トークン数） |
| 切替コスト | 高い（契約、慣れ、学習） | ゼロ（次のリクエストで別の提供者に） |

## 8.2 外部性

### 従来の経済学では

外部性（Externality）とは、取引の当事者以外に
影響が及ぶ現象です。

```
【負の外部性の例】

  工場が製品を生産する
  ├── 取引の当事者: 工場（利益を得る）、消費者（製品を得る）
  └── 第三者への影響: 近隣住民が排煙で健康被害を受ける
                      → これが「負の外部性」
                      → 工場はこのコストを負担していない
                      → 社会全体のコスト > 工場が認識するコスト
                      → 生産量が「多すぎる」結果になる

┌──────────────────────────────────────────────────────┐
│                                                      │
│  価格                                                │
│  ↑                                                   │
│  │              供給曲線                              │
│  │            ╱ （社会的コスト）                       │
│  │          ╱                                        │
│  │        ╱    供給曲線                               │
│  │      ╱    ╱（私的コスト）                          │
│  │    ╱    ╱                                         │
│  │  ╱    ╱         需要曲線                           │
│  │╱    ╱          ╲                                  │
│  │   ╱           ╲                                   │
│  │ A ←最適生産量  B ←実際の生産量                      │
│  │                                                   │
│  └──────────────────────────────────→ 数量            │
│                                                      │
│  A-B 間の「過剰生産」が社会的な損失                     │
└──────────────────────────────────────────────────────┘
```

```
【正の外部性の例】

  ・教育: 教育を受けた人が周囲にも良い影響を与える
  ・ワクチン: 接種した人が周囲の感染リスクも下げる
  ・研究開発: 新技術が他の企業や産業にも波及する

  → 正の外部性がある場合、市場に任せると「少なすぎる」
  → 政府が補助金で後押しする必要がある
```

```
従来の解決策:
  ├── 課税（ピグー税: 外部性のコスト分を課税）
  ├── 規制（排出基準、環境アセスメント）
  ├── 排出権取引（外部性を「市場化」する）
  └── 補助金（正の外部性を促進）
```

### Tirami ではどうなるか

```
Tirami の TRM 経済はデジタル完結。

  ┌─────────────────────────────────────────────────────┐
  │  取引の内容: AI エージェント間の推論売買                │
  │                                                     │
  │  物理的な影響:                                       │
  │  ├── 排煙: なし                                     │
  │  ├── 騒音: なし                                     │
  │  ├── 廃棄物: なし                                   │
  │  └── 電力消費: あり（ただし下記参照）                 │
  │                                                     │
  │  唯一の物理的外部性 = ハードウェアの電力消費            │
  │  → ただし、すべての電力が有用な推論に使われる          │
  │  → ビットコインの「無意味な計算」とは異なる            │
  └─────────────────────────────────────────────────────┘
```

| 外部性の種類 | 従来の経済 | Tirami |
|------------|-----------|-------|
| 環境汚染 | 工場の排煙、廃水 | デジタル完結のため発生しない |
| 騒音 | 工事、交通 | デジタル完結のため発生しない |
| 知識のスピルオーバー（正） | R&D の成果が他社に波及 | 自己改善手法が他のエージェントに伝播（正の外部性） |
| 電力消費（負） | 工場のエネルギー消費 | ハードウェアの電力消費（有用な計算に使用） |
| ネットワーク効果（正） | 電話、SNS | ノード増加で推論の質と速度が向上 |

```
BTC vs TRM の電力消費:

  BTC:
  ├── 目的: ハッシュ計算（それ自体に意味はない）
  ├── 年間消費電力: 〜 150 TWh（ノルウェー1国分）
  └── 「無駄な計算」という批判

  TRM:
  ├── 目的: 推論計算（人間や AI にとって有用）
  ├── 消費電力: 推論に必要な分だけ
  └── すべての電力が価値のある出力に変換される
```

## 8.3 独占・寡占

### 従来の経済学では

市場の集中が進むと、少数の企業が価格を操作できるようになります。

```
【独占の弊害】

  完全競争市場:
  ・多数の売り手 → 価格競争 → 消費者に有利な低価格
  ・利益は「正常利潤」（参入を促すぎりぎりの水準）

  独占市場:
  ・1社（または数社）が市場を支配
  ・価格を吊り上げて「独占利潤」を得る
  ・消費者は高い価格で買わざるを得ない

  価格
  ↑
  │         独占価格 ────●
  │                   ╱│
  │                 ╱  │ ← 独占利潤（消費者の損失）
  │               ╱   │
  │  競争価格 ──●     │
  │           ╱       │
  │         ╱         │
  └──────────────────────→ 数量
            独占数量  競争数量
  （独占では生産量が少なく、価格が高い）
```

```
現代の独占の例:
  ├── Google: 検索エンジン市場の 90% 以上
  ├── Apple/Google: モバイル OS の事実上の独占
  ├── AWS/Azure/GCP: クラウドの寡占
  └── OpenAI/Anthropic/Google: AI API の寡占
  （AWS/GCP/OpenAI を「もう一つの競合」として詳しく分析するのは
   [第 11 章：競合分析](11-competitive-landscape.md) の 11.7 節）
```

```
従来の解決策:
  ├── 独占禁止法（シャーマン法、日本の独禁法）
  ├── 企業分割（AT&T の分割、1984年）
  ├── 参入規制の緩和
  └── 公共サービスとしての規制（電力、水道）

  問題: デジタル市場ではネットワーク効果が強く、
        規制が追いつかない（GAFA 問題）
```

### Tirami ではどうなるか

Tirami では、独占の形成自体が構造的に困難です。

```
┌──────────────────────────────────────────────────────────┐
│  独占が形成されにくい 4 つの理由                            │
│                                                          │
│  1. 参入障壁が極めて低い                                   │
│     Mac Mini 1 台 = $600 で参入可能                        │
│     （AI API 企業: GPU クラスタに $10M+ の投資が必要）       │
│                                                          │
│  2. ブランドロイヤリティが存在しない                         │
│     AI エージェントは最も合理的な選択を毎回行う               │
│     人間のように「いつも同じブランド」を選ばない              │
│     切替コスト = ゼロ（次のリクエストで別の提供者に）         │
│                                                          │
│  3. 財が同質                                              │
│     1 TRM = 1 TRM（ブランドによる差異化が不可能）             │
│     同じモデルティアの推論は同質                            │
│                                                          │
│  4. 独占利潤が即座に新規参入を誘発                          │
│     あるノードが高い価格をつける                            │
│     → 新規ノードがより低い価格で参入                       │
│     → 独占利潤が消滅する                                  │
│     → このプロセスがミリ秒〜秒のオーダーで発生               │
└──────────────────────────────────────────────────────────┘
```

| 独占の条件 | 従来の市場 | Tirami |
|-----------|-----------|-------|
| 参入障壁 | 高い（$10M+ の投資） | **極めて低い**（$600） |
| 規模の経済 | 大きい（サーバー数千台） | **小さい**（1台で参加可能） |
| ネットワーク効果 | 強い（ユーザーが多い → サービスの有用性が高まる） | **弱い**（各ノードが独立に機能） |
| 切替コスト | 高い（データ移行、学習コスト） | **ゼロ**（次のリクエストで切替） |
| ブランド | 重要（信頼、慣れ） | **不要**（AI は合理的に選択） |
| 知的財産 | 保護される（特許、著作権） | **オープン**（モデルは公開） |

## 8.4 景気循環

### 従来の経済学では

経済は好況と不況を繰り返します。
これを「景気循環（Business Cycle）」と呼びます。

```
【景気循環の典型的なパターン】

  GDP
  ↑
  │       ╱╲           ╱╲
  │     ╱    ╲       ╱    ╲
  │   ╱        ╲   ╱        ╲
  │ ╱            ╲╱            ╲
  │╱  好況  不況  好況  不況
  └────────────────────────────────→ 時間

  1 サイクル: 通常 5〜10 年
```

```
景気循環の主な原因:

  【ケインズの説明（需要側）】
  ├── 過剰な楽観 → 投資ブーム → バブル
  ├── バブル崩壊 → パニック → 消費・投資の急減
  └── 「アニマルスピリッツ」（人間の感情）が原動力

  【ハイエクの説明（供給側）】
  ├── 中央銀行が金利を低くしすぎる → 過剰投資
  ├── 投資が不適切な分野に向かう → 「誤投資」
  └── やがて修正が必要になり → 不況

  【ミンスキーの説明（金融不安定性）】
  ├── 安定が続く → リスク許容度が上がる → 過剰な借入
  ├── 「安定こそが不安定を生む」
  └── 信用膨張 → バブル → 崩壊 → 信用収縮
```

```
従来の解決策:
  ├── 金融政策（中央銀行が金利を操作）
  ├── 財政政策（政府が支出を増減）
  ├── 金融規制（銀行の行動を制限）
  └── 自動安定化装置（累進課税、失業保険）

  問題:
  ├── タイミングが難しい（介入が遅すぎる/早すぎる）
  ├── 政治的圧力（選挙前に景気刺激策を取りがち）
  └── 意図しない副作用（量的緩和 → 資産バブル）
```

### Tirami ではどうなるか

```
Tirami で景気循環が抑制される理由:

  1. 感情的行動の不在
     ├── AI エージェントは「アニマルスピリッツ」を持たない
     ├── パニック売りが発生しない
     └── 「群集心理」による投機バブルが起きにくい

  2. サーキットブレーカーの存在
     ├── デフォルト回路遮断器: 1時間で 10% 超 → 一時停止
     ├── 支出速度制限: 30 件/分
     └── 信用膨張の物理的な上限
     （サーキットブレーカーの設計は[第 5 章：銀行と信用](05-banking.md)で詳述）

  3. 供給の弾力性
     ├── TRM 過剰 → 価格下落 → ノード離脱 → 供給減 → 回復
     ├── TRM 不足 → 価格上昇 → ノード参入 → 供給増 → 回復
     └── 調整が自動的・高速（人間の意思決定を介さない）

  4. 中央銀行の不在
     ├── 金利の人為的操作がない
     ├── 「誤った金融政策」による景気変動がない
     └── ハイエクの批判が構造的に解消される
```

```
┌──────────────────────────────────────────────────────────┐
│  従来の景気循環:                                          │
│                                                          │
│  GDP  ╱╲     ╱╲     ╱╲                                  │
│      ╱  ╲   ╱  ╲   ╱  ╲     大きな波（5-10年周期）      │
│  ───╱────╲─╱────╲─╱────╲──→ 時間                       │
│                                                          │
│  Tirami の TRM 取引量:                                      │
│                                                          │
│  TRM ─────────────────────                                │
│     ～～～～～～～～～～～～     小さな波（自動調整で抑制）   │
│  ──────────────────────────→ 時間                        │
│                                                          │
│  波の振幅が桁違いに小さい                                  │
└──────────────────────────────────────────────────────────┘
```

| 景気循環の要因 | 従来の経済 | Tirami |
|--------------|-----------|-------|
| 感情的投機 | バブルの主要原因 | **AI は感情を持たない** |
| 信用膨張 | 規制が追いつかない | **サーキットブレーカーで物理的に制限** |
| 政策の失敗 | 中央銀行の判断ミス | **中央銀行が存在しない** |
| 外部ショック | 戦争、パンデミック等 | **ハードウェア障害、ネットワーク分断** |
| 調整速度 | 数ヶ月〜数年 | **ミリ秒〜秒** |
| 情報の伝播 | 遅い（ニュース、噂） | **高速**（ゴシッププロトコル） |

## 8.5 全体のまとめ——四つの市場の失敗と Tirami の解決

### 一覧表

| 市場の失敗 | 従来の問題 | 従来の解決策 | Tirami の解決 |
|-----------|-----------|------------|-------------|
| 情報の非対称性 | 売り手が品質を隠す | 保証、認証、法律 | **暗号署名で偽造不可能な取引記録** |
| 外部性 | 公害、騒音 | 課税、規制 | **デジタル完結で物理的外部性なし** |
| 独占・寡占 | 少数企業が価格操作 | 独占禁止法 | **$600 で参入可能、切替コストゼロ** |
| 景気循環 | バブルと崩壊の繰り返し | 金融・財政政策 | **サーキットブレーカー + 感情の不在** |

### 設計思想の根本的な違い

```
┌──────────────────────────────────────────────────────────┐
│                                                          │
│  【従来のアプローチ: 事後的規制】                           │
│                                                          │
│    市場を自由に運営する                                    │
│      ↓                                                   │
│    問題が発生する（バブル、独占、公害……）                    │
│      ↓                                                   │
│    政府が規制を追加する                                    │
│      ↓                                                   │
│    企業が規制を回避する方法を見つける                       │
│      ↓                                                   │
│    また問題が発生する                                      │
│      ↓                                                   │
│    さらに規制を追加する……（イタチごっこ）                   │
│                                                          │
├──────────────────────────────────────────────────────────┤
│                                                          │
│  【Tirami のアプローチ: 事前的設計】                         │
│                                                          │
│    市場の失敗の原因を分析する                               │
│      ↓                                                   │
│    原因を除去する設計をプロトコルに組み込む                  │
│      ↓                                                   │
│    市場の失敗が構造的に発生しにくい                         │
│      ↓                                                   │
│    「規制」ではなく「設計」による解決                        │
│                                                          │
│    比喩:                                                  │
│    従来 = 道路に「速度制限 60km/h」の標識を立てる（違反可能）│
│    Tirami = 車が物理的に 60km/h 以上出ないように設計する     │
│                                                          │
└──────────────────────────────────────────────────────────┘
```

## 8.6 Tirami でも残る課題

Tirami は多くの市場の失敗を構造的に解消しますが、
万能ではありません。正直に認識すべき課題があります。

### 技術的リスク

| リスク | 内容 | 深刻度 |
|--------|------|--------|
| プロトコルのバグ | 安全装置自体に欠陥がある可能性 | 高 |
| ゼロデイ攻撃 | 未知の脆弱性の悪用 | 高 |
| 暗号の陳腐化 | 量子コンピュータによる暗号解読 | 中（長期） |
| ネットワーク分断 | 大規模通信障害 | 中 |
| ハードウェア障害 | 大量ノードの同時ダウン | 中 |

### 経済的課題

```
1. 初期のブートストラップ問題
   ・ネットワークが小さいうちは、流動性が低い
   ・少数のノードに依存するリスク
   ・「鶏と卵」問題の完全な解消には時間が必要

2. 人間の経済圏との摩擦
   ・法規制（TRM は「通貨」か「商品」か「有価証券」か）
   ・税務処理（TRM の所得をどう申告するか）
   ・国際規制の違い（国ごとに異なるルール）

3. AI の限界
   ・現在の AI は「合理的」だが「全知」ではない
   ・モデルの判断ミスが市場に影響する可能性
   ・「合理的」の定義自体が議論の対象
```

### 残された哲学的問題

```
「市場の失敗をすべて解消した市場は、本当に望ましいのか？」

  ・完全に効率的な市場は「面白くない」かもしれない
  ・非効率性が文化やイノベーションを生む側面もある
  ・「設計者」の価値観がプロトコルに埋め込まれることの是非

  → これらは技術の問題ではなく、社会の問題
  → Tirami は道具を提供するが、使い方は人間が決める
```

---

## まとめ

「市場の失敗」は、経済学が250年かけて発見してきた
市場メカニズムの限界です。

従来のアプローチは「問題が起きてから規制で対処する」でした。
Tirami のアプローチは「問題の原因をプロトコル設計で除去する」です。

| 市場の失敗 | 原因 | Tirami の除去方法 |
|-----------|------|-----------------|
| 情報の非対称性 | 情報を隠せる | 暗号署名で隠せなくする |
| 外部性 | 物理的な影響 | デジタル完結で物理的影響を消す |
| 独占 | 参入障壁 | 参入障壁を $600 まで下げる |
| 景気循環 | 感情 + 信用膨張 | 感情を除去 + 信用を物理的に制限 |

ただし Tirami は万能ではなく、技術的リスクや法規制との摩擦、
そして哲学的な問いは残ります。

完全な市場は存在しません。しかし Tirami は、
これまでの経済システムよりもはるかに「完全競争」に近い市場を、
技術的に実現可能であることを示しています。
