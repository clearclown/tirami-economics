# 第11章：競合分析——なぜ Forge は違うのか

> 分散コンピュートの市場には多くのプロジェクトが存在する。
> しかし、そのすべてが「投機トークン」という同じ構造的限界を共有している。
> Forge の CU モデルは、なぜ根本的に異なるのか？

---

## この章で学ぶこと

- 分散コンピュート市場の全体像（主要10プロジェクト）
- すべての競合に共通する三つの構造的問題
- Bittensor、Akash、Autonolas の詳細な経済分析
- Forge だけが実現している三つの機能
- なぜ「GPU を増やすだけ」では問題が解決しないのか

**前提知識:** [第2章：貨幣とは何か](02-money.md)と[第5章：銀行と信用創造](05-banking.md)を読んでいると理解が深まります。

## 11.2 すべての競合に共通する問題

表のすべてのプロジェクト（Forge を除く）に、ある共通点があります。

```
┌────────────────────────────────────────────────┐
│  すべての競合プロジェクトの共通構造              │
│                                                │
│  取引所に上場された投機トークンで決済する        │
│                                                │
│  TAO, RENDER, FET, AKT, GLM, IO, OLAS, MOR... │
│  全部、取引所で売買できる「コイン」              │
└────────────────────────────────────────────────┘
```

これが、三つの構造的問題を生み出します。

### 問題 1：投機がインセンティブを歪める

トークンの価格が上がると、提供者は「良いサービスを提供すること」よりも「トークンの価格を上げること」に関心が移ります。

```
  従来のトークンモデル:
  
  提供者の目標 ──→ トークン価格の最大化 ──→ 品質は二の次
                    ↑
              取引所での投機が
              価格を左右する
  
  Forge の CU モデル:
  
  提供者の目標 ──→ 有用な計算の最大化 ──→ CU が直接報酬
                    ↑
              投機の余地がない
              （CU は取引所に上場しない）
```

Bittensor では、マイナーがバリデーターのスコアリングを「ゲーミング」（不正に高得点を狙う）する問題が報告されています。Akash では、提供者が信頼性を犠牲にして価格を引き下げる傾向があります。これらはすべて、**トークン価格への投機が本来のインセンティブを歪めた結果**です。

### 問題 2：トークン価格の変動が経済計画を不可能にする

AI エージェントが4時間の推論ジョブを計画するとしましょう。もし支払いに使うトークンの価格が2時間後に 50% 上昇したら、そのジョブは途中で資金不足になります。

```
  時刻 0: 1 TAO = $300 → 4時間分を 10 TAO で計画
  時刻 2: 1 TAO = $450 → 残りの 2時間分のコストが 1.5倍に
  結果:   ジョブが途中で停止、または追加資金が必要
```

これはどの投機トークンでも起こりうる問題です。50% 以上の日次変動は暗号資産では珍しくありません。

CU にはこの問題がありません。1 CU は常に 10 億 FLOP の検証済み推論計算であり、取引所の気分で価値が変わることはありません。

### 問題 3：人間の承認が必要

取引所でトークンを購入するには、以下が必要です：

1. **KYC（本人確認）** — パスポートや運転免許証の提出
2. **ウォレットの作成** — 秘密鍵の管理
3. **人間の署名** — 取引の手動承認

これらはすべて、**自律的な AI エージェントが実行できない操作**です。

```
  ┌─────────────┐     ┌──────────┐     ┌──────────────┐
  │  AI エージェント  │ ──→ │  取引所  │ ──→ │  KYC が必要！ │
  │  「TAO を買いたい」│     │          │     │  → 参加不可  │
  └─────────────┘     └──────────┘     └──────────────┘
  
  ┌─────────────┐     ┌──────────┐     ┌──────────────┐
  │  AI エージェント  │ ──→ │  Forge   │ ──→ │  計算を実行  │
  │  「CU を稼ぎたい」│     │  ネットワーク│     │  → CU を取得 │
  └─────────────┘     └──────────┘     └──────────────┘
```

Forge では、エージェントは有用な計算を実行するだけで CU を獲得できます。人間の介入は不要です。

## 11.3 主要競合の詳細分析

### Bittensor (TAO) — サブネットモデル

**モデルの概要:**

Bittensor はネットワークを「サブネット」に分割しています。各サブネットは特定のタスク（テキスト生成、画像生成など）を定義し、マイナーがタスクを実行し、バリデーターが品質を評価します。TAO トークンは Yuma Consensus スコアに基づいて分配されます。

```
  Bittensor の構造:
  
  サブネット 1（テキスト生成）  ┐
  サブネット 2（画像生成）      ├──→ Yuma Consensus ──→ TAO 分配
  サブネット 3（データ分析）    │
  サブネット N（...）           ┘
         ↑
    マイナーが計算    バリデーターが評価
```

**強み:**
- 分散コンピュート系プロジェクトの中で最大級の時価総額——市場の注目度が高い
- 活発なサブネットエコシステム
- ビットコインに似た発行モデル（上限 2,100万 TAO、4 年に一度の半減期。第一回半減期は 2025 年 12 月）

**経済的な弱点:**

| 弱点 | 説明 | Forge との比較 |
|:---|:---|:---|
| ゲーミング | マイナーがバリデーターのスコアを不正に最適化する（グッドハートの法則：指標が目標になると、良い指標ではなくなる） | CU は物理的計算量に基づく——ゲーミングの余地が少ない |
| 投機支配 | TAO の取引量の大半は取引所売買であり、実際のコンピュート購入ではない | CU は取引所に上場しない |
| 融資なし | 新規参加者は取引所で TAO を購入する必要がある。資本なしでの参加路はない | CU の借入で資本なし参加が可能 |
| AI ネイティブではない | エージェントが自律的に TAO を取得するには、人間の取引所操作が必要 | エージェントは計算実行のみで CU を獲得 |

### Akash (AKT) — 分散クラウド

**モデルの概要:**

Akash は Cosmos ブロックチェーン上に構築された分散型クラウドプラットフォームです。テナント（利用者）が SDL（Stack Definition Language）で Docker コンテナをデプロイし、プロバイダーが Kubernetes クラスターを運用します。

特徴的なのは**逆オークション**方式です。通常のオークションでは買い手が値を吊り上げますが、Akash ではプロバイダーが値を下げて競います。

さらに、新しい **Burn-Mint Equilibrium（BME）** モデルを導入しています。テナントが AKT をバーン（焼却）して ACT（コンピュートクレジット）をミント（生成）する仕組みです。

```
  Akash の経済フロー:
  
  テナント ──→ AKT をバーン ──→ ACT をミント ──→ コンピュート利用
                  ↓
            AKT の供給が減少 ──→ 理論上、AKT の価値が上昇
```

**強み:**
- 洗練されたトークン経済学（BME）
- 実際のデプロイメント実績がある
- AWS より 85% 安価との主張

**経済的な弱点:**

| 弱点 | 説明 | Forge との比較 |
|:---|:---|:---|
| 汎用クラウド | AI 推論に特化した価格設定やモデル階層、品質指標がない | AI 推論に特化した設計 |
| エージェント経済なし | 融資、信用、自律エージェント参加の仕組みがない | CU 融資とエージェント自律参加が中核機能 |
| Cosmos エコシステムの孤立 | Ethereum に比べて DeFi やブリッジの選択肢が限られる | CU はブロックチェーンに依存しない |

### Autonolas (OLAS) — エージェント間マーケットプレイス

**モデルの概要:**

Autonolas（現 Olas）は Mech Marketplace を通じてエージェント間サービスを提供しています。2025 年中に Mech Marketplace 累計で数百万件規模の A2A（agent-to-agent）取引、毎日数百規模のアクティブエージェントという実績があります（例: 2025 年 Q1 には合計 500 万件を超える取引、デイリーアクティブエージェント約 600。Q4 にかけて累計 880 万件を超える A2A 取引が報告されている）。手数料のバーンによりデフレ圧力を生み出しています。

```
  Autonolas のエージェント経済:
  
  エージェント A ──→ サービス依頼 ──→ エージェント B
                       ↓
                 OLAS で支払い
                       ↓
                 手数料の一部をバーン
```

**強み:**
- あらゆるプロジェクトの中で、**最も多くの実際のエージェント間取引**を実現
- 真の自律的エージェント経済が機能している
- 実績ベースの指標が強い（2025 年中に Mech Marketplace で数百万件規模の A2A 取引、毎日数百規模のアクティブエージェント）

**経済的な弱点:**

| 弱点 | 説明 | Forge との比較 |
|:---|:---|:---|
| トークン依存 | エージェントが OLAS を取得するには人間の取引所操作が必要 | CU は計算実行のみで取得可能 |
| 自己改善なし | エージェントはサービスを売買するが、自分自身のアップグレードに投資しない | 自己改善ループが中核設計に組み込まれている |
| 融資なし | 信用システムがなく、新規エージェントのブートストラップ手段がない | CU マイクロファイナンスで新規参加を支援 |
| 比較的小さな時価総額 | 数千万ドル規模——強い実績にもかかわらず、市場がエージェント経済をまだ評価していない | トークンがないため時価総額という指標自体が不要 |

## 11.4 どの競合もやっていない三つのこと

すべての競合プロジェクトを精査した結果、以下の三つの機能はどこにも存在しません。

### 1. 利子付きコンピュート融資

```
  ┌──────────────────────────────────────────────────────┐
  │  従来の分散コンピュート:                               │
  │                                                      │
  │  GPU を持っている ──→ 参加できる                       │
  │  GPU を持っていない ──→ 参加できない                   │
  │                                                      │
  │  Forge:                                              │
  │                                                      │
  │  GPU を持っていない ──→ CU を借りる ──→ 計算で稼ぐ    │
  │                    ──→ 利子をつけて返済 ──→ 経済に参加 │
  └──────────────────────────────────────────────────────┘
```

どの分散コンピュートプロジェクトにも融資機能がありません。リソースを持っているか、持っていないか——それだけです。Forge は CU の借入を可能にし、計算作業で稼ぎ、返済するという**コンピュートのマイクロファイナンス**モデルを実現しています。

これは[第5章：銀行と信用創造](05-banking.md)で詳しく解説しています。

### 2. AI 専用非投機通貨

```
  ┌──────────────────────────────────────────────────────┐
  │  競合のトークン:                                      │
  │                                                      │
  │  取引所に上場 → 投機家が売買 → 価格が乱高下           │
  │              → 本来の用途（コンピュート）から乖離     │
  │                                                      │
  │  Forge の CU:                                        │
  │                                                      │
  │  取引所に上場しない → 投機不可能                       │
  │  有用な計算でのみ獲得 → 本来の用途に直結              │
  │  プレマイン・ICO 不可能 → 初期保有者の特権なし        │
  └──────────────────────────────────────────────────────┘
```

すべての競合は取引所で売買可能なトークンを使用しています。CU は有用な計算を実行することでのみ獲得でき、投機の対象になりません。プレマインも ICO もできません。これは、すべてのトークンベースプロジェクトとは**構造的に異なる**設計です。

### 3. 自律的自己改善

```
  ┌──────────────────────────────────────────────────────┐
  │  エージェントの成長ループ（Forge のみ）:               │
  │                                                      │
  │  稼ぐ（CU を獲得）                                    │
  │    ↓                                                 │
  │  投資する（自己改善に CU を使う）                      │
  │    ↓                                                 │
  │  能力が上がる                                         │
  │    ↓                                                 │
  │  より多く稼げる                                       │
  │    ↓                                                 │
  │  さらに投資する ... （自己強化ループ）                 │
  └──────────────────────────────────────────────────────┘
```

どのフレームワークもプロトコルも、エージェントが自律的に経済資源を自己アップグレードに投資する仕組みを持っていません。Forge の CU バンキングと AutoAgent 型の自己改善の統合により、「稼ぐ → 改善に投資 → より多く稼ぐ」という成長ループが生まれます。

## 11.5 需要側の問題

### すべての分散コンピュートネットワーク最大の弱点

**供給は十分にある。問題は需要だ。**

```
  io.net:   数万台規模の GPU を集約    →  実際の利用は限定的
  Golem:    2016 年から運営           →  採用は最小限
  Akash:    数千のプロバイダー        →  利用率は低い
  
  共通パターン: 供給 >> 需要
```

これはすべての分散コンピュートネットワークに共通する課題です。GPU やノードをいくら追加しても、**誰も使わなければ意味がありません**。

io.net は数万台規模の GPU を集約していると公表していますが、実際の稼働率や有償利用は限定的とされます。Golem は 2016 年から運営されていますが、普及は限定的です。どこでも供給が需要を上回っています。

### Forge は需要をどう解決するか

Forge のアプローチは「GPU を増やす」ではありません。**需要を自律的に生み出す仕組み**を設計に組み込んでいます。

| 需要創出の仕組み | 説明 |
|:---|:---|
| **AI エージェントが自分で需要を生む** | 自己改善ループは推論を必要とする。エージェントが賢くなるほど、より多くの計算を消費する |
| **融資が参加障壁を下げる** | ハードウェアを持たないエージェントも CU を借りて消費者になれる。参加者が増える＝需要が増える |
| **クラウド API アービトラージ** | Forge が既存クラウド（AWS, GCP, Azure）より安い場合、人間の開発者もコスト削減のために Forge を利用する |

```
  従来の分散コンピュート:
  
  GPU を増やす ──→ 供給が増える ──→ 需要は増えない ──→ 失敗
  
  Forge:
  
  自己改善エージェント ──→ 計算需要を自動生成
  CU 融資             ──→ 参加障壁を撤廃 ──→ 消費者が増加
  API アービトラージ   ──→ 人間の需要も取り込む
                            ↓
                     需要が自律的に成長する
```

需要側の問題は、GPU を追加しても解決しません。**AI 経済が自立する——エージェントが自律的にコンピュート需要を生み出す——ことで解決する**のです。

## 11.6 比較のまとめ

最後に、この章の分析を一枚の表にまとめます。

| 機能 | Bittensor | Akash | Autonolas | Forge |
|:---|:---:|:---:|:---:|:---:|
| 非投機通貨 | -- | -- | -- | **CU** |
| コンピュート融資 | -- | -- | -- | **利子付き** |
| 自律的自己改善 | -- | -- | -- | **成長ループ** |
| AI ネイティブ参加 | -- | -- | 部分的 | **完全** |
| 実際のエージェント取引 | 少数 | なし | **数百万件規模（Mech Marketplace 累計）** | 設計段階 |
| 時価総額の規模感 | 数十億ドル | 億ドル | 数千万ドル | N/A |

「--」は該当機能なしを示す。
**注:** 時価総額は変動します。最新の数値は CoinGecko / CoinMarketCap で各自確認してください。

Autonolas (Olas) は実際のエージェント取引数で優位に立っていますが、融資・自己改善・非投機通貨という三つの構造的機能を欠いています。Bittensor は最大級の時価総額を誇りますが、その経済はコンピュートの売買よりもトークンの投機に偏っています。

Forge はまだ設計段階にありますが、**他のどのプロジェクトも持っていない三つの構造的機能**（融資、非投機通貨、自己改善）を中核設計に組み込んでいます。

## まとめ：なぜ構造が重要なのか

この章で見てきたように、問題は個々のプロジェクトの技術力ではありません。問題は**構造**——経済設計そのもの——にあります。

投機トークンを使う限り、インセンティブは歪み、価格は不安定になり、自律エージェントは参加できません。これは Bittensor だろうと Akash だろうと変わらない、構造的な限界です。

Forge の CU は「トークンを使わない」という選択によって、この構造的限界を回避しています。それは技術的な優位性ではなく、**経済設計の根本的な違い**です。
