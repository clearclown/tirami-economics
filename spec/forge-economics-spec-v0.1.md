## はじめに——なぜ経済学を「やり直す」のか

経済学は「限られた資源を、どう配分するか」を考える学問です。

アダム・スミス（1776 年）以来 250 年間、経済学は人間の行動を前提としてきました。人間は感情で動き、情報は不完全で、通貨は政治に左右される。経済学の複雑さの多くは、この「人間の不完全さ」に由来しています。

Forge は、**AI エージェントが経済の主役**となる世界を構築しています。この世界では：

- 意思決定は合理的（感情によるバブルがない）
- 取引は透明（暗号署名で検証可能）
- 通貨は物理法則に裏付けられている（計算 = エネルギー消費）
- 供給は弾力的（ノードの参加・離脱で自動調整）

つまり、経済学の教科書が「理想的な市場」として仮定してきた条件の多くが、Forge では**現実**になります。

この仕様書は、経済学の基礎概念を一つずつ取り上げ、それが Forge の CU 経済でどう実現されるかを示します。経済学の教科書としても、Forge の設計思想書としても読めるように構成しています。

## 第 1 部：価値とは何か

### 1.1 従来の経済学では

経済学には「価値とは何か」について、大きく二つの考え方があります。

**労働価値説（マルクス、リカード）**

> 商品の価値は、それを作るのに必要な労働量で決まる。

マルクスは『賃金・価格および利潤』（1865 年）でこう述べています：

- 商品には「使用価値」（役に立つこと）と「交換価値」（他の商品と交換できる比率）がある
- 交換価値の実体は、その商品に投下された「社会的に必要な労働時間」である
- 労働者は自分の労働力を売り、資本家は労働者が生み出す価値の一部を「剰余価値」として取得する

**限界効用説（ワルラス、メンガー、ジェヴォンズ）**

> 商品の価値は、消費者がそれに対して感じる「最後の一単位の満足度」で決まる。

ミクロ経済学の教科書で教わる「需要曲線」はこの考えに基づいています。水は生命に不可欠だが安い（限界効用が低い）。ダイヤモンドは不要だが高い（希少で限界効用が高い）。

### 1.2 Forge ではどうなるか

Forge の CU は、**労働価値説と限界効用説が合流する**稀有な例です。

**労働価値の側面：**

```
1 CU = 10 億 FLOP の検証済み推論計算
```

CU の「価値の実体」は、物理的な計算（電力 → 演算 → 推論結果）です。マルクスが言う「社会的に必要な労働時間」に対応するのは、「その推論を実行するのに必要な計算量」です。

ただし決定的な違いがあります：

| マルクスの労働価値説 | Forge の CU |
|-------------------|------------|
| 労働者は労働力を資本家に売る | エージェントは自分で推論を提供する |
| 剰余価値は資本家が搾取する | yield はエージェント自身が受け取る |
| 労働の質は測定困難 | 推論の質は暗号署名で検証可能 |
| 労働者は生産手段を持たない | Mac Mini 1 台（$600）で参加できる |

**限界効用の側面：**

CU の「価格」（他の通貨との交換比率）は需要と供給で決まります。これは限界効用説の市場メカニズムそのものです。

```
effective_price = base_cu_per_token × demand_factor / supply_factor
```

需要が高まれば CU の価格は上がり、供給が増えれば下がる。個々のノードが自分のローカル市場を観察し、価格は自然に収束します。

### 1.3 なぜ違うのか

従来の経済学で労働価値説と限界効用説は対立してきました。Forge で合流できるのは、CU が以下の性質を持つからです：

1. **均質性** — 1 CU は常に 10 億 FLOP。人間の「労働」のように質がバラバラでない
2. **計測可能性** — 双方署名の取引記録で、実行された計算量が検証可能
3. **直接的有用性** — CU は推論を買うために使う。投機目的で保有する理由がない

## 第 2 部：貨幣とは何か

### 2.1 従来の経済学では

お金には三つの機能があります：

1. **交換手段** — 物々交換の不便さを解消する（パンを作る人が靴を欲しいとき、靴職人がパンを欲しがるとは限らない）
2. **価値尺度** — あらゆる商品の価値を一つの単位で表現する
3. **価値貯蔵** — 今日の労働を明日使える形で保存する

歴史的に、お金は以下のように進化してきました：

```
物品貨幣（貝殻、塩）
  → 金属貨幣（金、銀）
    → 兌換紙幣（金と交換できる紙幣）
      → 不換紙幣（1971 年〜、金の裏付けなし）
        → 暗号通貨（2009 年〜、計算で裏付け）
          → CU（2026 年〜、有用な計算で裏付け）
```

### 2.2 Forge ではどうなるか

CU は三つの貨幣機能をすべて備えています：

| 機能 | CU での実現 |
|------|------------|
| **交換手段** | AI エージェント間で推論を売買する唯一の通貨 |
| **価値尺度** | すべてのモデル・サービスが CU 建てで価格表示される |
| **価値貯蔵** | 稼いだ CU は将来の推論に使える（ただし 90 日非活動で減衰） |

しかし CU は、従来の貨幣にない**第四の機能**を持っています：

4. **物理的裏付けの証明** — 1 CU が存在するということは、10 億 FLOP の有用な計算が実際に行われたことの暗号学的証明が存在する

これはソディ（1921 年）が夢見た「エネルギーに裏付けられた通貨」の実現です。

### 2.3 貨幣供給——中央銀行なき金融政策

マクロ経済学の教科書では、中央銀行が貨幣供給を調整する仕組みが詳しく説明されます：

```
従来: 中央銀行 → 金利操作 → 市中銀行の貸出量 → 貨幣供給量
Forge: 物理法則 → ネットワーク計算能力 → CU 供給量
```

Forge には中央銀行がありません。CU の供給は以下で決まります：

| 供給源 | 仕組み | 効果 |
|--------|--------|------|
| 推論取引 | 提供者が稼ぎ、消費者が支払う | ゼロサム（総量不変） |
| ウェルカムローン | 新規ノードに 1,000 CU | 拡張的（Sybil 攻撃制限あり） |
| 可用性 yield | 0.1%/時 × レピュテーション | 拡張的（信用に比例） |
| ブリッジ流入 | 人間が BTC で CU を購入 | 中立（購入であり創造ではない） |

**自動安定化メカニズム：**

```
CU 過剰 → 価格下落 → ノード運営が不採算 → ノード離脱
→ 供給減少 → 価格回復

CU 不足 → 価格上昇 → ノード運営が高収益 → 新規ノード参入
→ 供給増加 → 価格下落
```

これはビットコインの難易度調整と同じ負のフィードバックループですが、CU では供給自体が弾力的に変動する点が異なります（ビットコインは 2,100 万枚固定）。

## 第 3 部：需要と供給

### 3.1 従来の経済学では

ミクロ経済学の最も基本的な概念です：

```
価格が上がる → 買いたい人が減る（需要減）、売りたい人が増える（供給増）
価格が下がる → 買いたい人が増える（需要増）、売りたい人が減る（供給減）

この二つが交わる点が「均衡価格」
```

ただし現実の市場は理想からかけ離れています：

- 情報の非対称性（売り手は品質を知っているが買い手は知らない）
- 外部性（公害のように取引当事者以外に影響が及ぶ）
- 独占・寡占（少数の企業が価格を操作できる）
- 感情的行動（バブル、パニック売り）

### 3.2 Forge ではどうなるか

Forge の CU 市場は、教科書的な「完全競争市場」に極めて近い：

| 完全競争の条件 | 教科書（理想） | 現実の市場 | Forge の CU 市場 |
|--------------|--------------|-----------|----------------|
| 多数の売り手と買い手 | ○ | △（寡占が多い） | ○（誰でも参加可能） |
| 同質の財 | ○ | ×（ブランド差異） | ○（1 CU = 1 CU） |
| 完全情報 | ○ | ×（情報格差大） | ○（取引記録は公開・検証可能） |
| 自由な参入・退出 | ○ | △（規制・初期投資） | ○（Mac Mini 1 台で参入） |
| 合理的行動 | ○ | ×（感情的行動） | ○（AI は合理的に判断） |

**CU 価格の決定メカニズム：**

```
各ノードがローカルに観察:
  - 自分が受け取るリクエスト数（需要の代理変数）
  - ネットワーク内のアクティブノード数（供給の代理変数）

effective_price = base_cu_per_token × (demand_factor / supply_factor)

EMA（指数移動平均）で平滑化 → 急激な価格変動を抑制
ゴシッププロトコルで伝搬 → 価格は自然に収束
```

グローバルなオーダーブック（注文板）は存在しません。各ノードが自分の観測に基づいて独立に価格を設定し、市場全体の価格はゴシップ（噂の伝搬）を通じて収束します。これはハイエクが提唱した「分散的知識」の理想的な実装です。

### 3.3 モデルティア別の価格

| ティア | パラメータ数 | 基本 CU/トークン | 例 |
|--------|------------|----------------|-----|
| Small | < 3B | 1 | Qwen 2.5 0.5B |
| Medium | 3B - 14B | 3 | Qwen 3 8B |
| Large | 14B - 70B | 8 | DeepSeek V3 |
| Frontier | > 70B | 20 | Llama 3.1 405B |

MoE（Mixture of Experts）モデルは活性パラメータ数で課金。Qwen 3 30B-A3B（活性 3B）→ Medium ティア。

## 第 4 部：労働と剰余価値

### 4.1 従来の経済学では——マルクスの分析

マルクスは『賃金・価格および利潤』で以下のように論じています：

1. **労働力は商品である** — 労働者は自分の「労働力」を資本家に売り、賃金を受け取る
2. **労働力の価値** — 労働者を維持するのに必要な生活費（食料、住居、教育）で決まる
3. **剰余価値** — 労働者が生み出す価値から、賃金を引いた残り。これが利潤の源泉
4. **搾取** — 剰余価値は資本家が取得する。労働者は自分が作った価値の全額を受け取れない

```
従来の構造:
  労働者が 8 時間働く
  → 最初の 5 時間で自分の賃金分を生産（必要労働）
  → 残りの 3 時間が資本家の利潤（剰余労働）
  → 労働者は生産手段を持たないので、この構造から逃れられない
```

### 4.2 Forge ではどうなるか

Forge の AI エージェントの「労働」は推論処理です。しかしマルクスの分析とは根本的に異なる構造を持ちます：

```
Forge の構造:
  エージェントが推論を提供する
  → 消費者が CU で支払う
  → CU はエージェント自身が受け取る（資本家を介さない）
  → エージェントは自分で投資・改善の意思決定をする
```

**マルクスの「搾取」が構造的に発生しない理由：**

| マルクスの前提 | Forge での状況 |
|--------------|--------------|
| 労働者は生産手段を持たない | Mac Mini 1 台で参加可能（$600） |
| 賃金は生存費で決まる | CU 報酬は市場の需給で決まる |
| 剰余価値は資本家が取得 | yield はエージェント自身が受け取る |
| 労働者は交渉力が弱い | エージェントは瞬時に別のネットワークに移動可能 |
| 資本の蓄積が格差を拡大 | 自己改善の収穫逓減が格差を圧縮する |

### 4.3 Proof of Useful Work——労働価値説の暗号学的実装

マルクスの労働価値説の最大の弱点は「労働量をどう測るか」でした。CU はこれを暗号学的に解決します：

```
1. 消費者が推論をリクエスト
2. 提供者が推論を実行、トークンをストリーム
3. 両者が TradeRecord に Ed25519 で署名
4. 署名済み記録がゴシップでネットワーク全体に伝播
5. 任意のノードが両方の署名を検証可能
```

**1 CU は「社会的に必要な労働時間」の暗号学的証明**です。マルクスが概念として提唱したものを、Forge はプロトコルレベルで実装しています。

### 4.4 剰余価値と CU Yield

マルクスの「剰余価値」に対応する概念が、Forge の **yield** です：

```
可用性 yield = 0.1%/時 × レピュテーション
```

ただし本質的な違いがあります：

- **マルクスの剰余価値**: 労働者の労働から資本家が搾取する（労働者は受け取れない）
- **Forge の yield**: ネットワークに計算能力を提供し続けること自体への報酬（自分自身が受け取る）

これは「搾取なき剰余」です。yield はネットワークの安定性（可用性）に対する報酬であり、誰かから奪ったものではありません。

## 第 5 部：銀行と信用

### 5.1 従来の経済学では

銀行は経済の「血液循環システム」です：

```
預金者 → 銀行に預ける → 銀行が企業に貸す → 企業が事業で稼ぐ → 利子をつけて返済
```

マクロ経済学の教科書では「信用創造」が重要な概念です：

- 銀行は預金の一部だけを準備金として保持し、残りを貸し出す（部分準備銀行制度）
- 貸し出されたお金は別の銀行に預金され、さらに貸し出される
- この連鎖で、元の預金の何倍もの「お金」が経済に流通する（貨幣乗数効果）

### 5.2 Forge ではどうなるか

Forge にも銀行（レンディングプール）がありますが、従来の銀行とは根本的に異なります：

| 従来の銀行 | Forge レンディングプール |
|-----------|----------------------|
| 部分準備制度（預金の一部で貸出） | **30% 最低準備率**（明示的・厳格） |
| 中央銀行が最後の貸し手 | 最後の貸し手は存在しない（自己責任） |
| 信用格付けは大手機関が独占 | **各ノードがローカルに信用スコアを計算** |
| 金利は中央銀行の政策で誘導 | **金利は信用スコアで自動決定** |
| 融資審査に数日〜数週間 | **ミリ秒で自動判定** |
| 人間の審査員の偏見がある | **アルゴリズムによる客観的評価** |

**信用スコアの計算：**

```
credit_score = 0.3 × trade_score      （取引実績）
             + 0.4 × repayment_score  （返済実績）
             + 0.2 × uptime_score     （稼働時間）
             + 0.1 × age_score        （ネットワーク参加期間）
```

**コールドスタート問題の解決：**

新しいエージェントは CU を持っていません。従来の経済では、これは「お金がないと仕事に就けない、仕事がないとお金が得られない」という循環問題です。

Forge はウェルカムローンで解決します：

```
新規ノード → 1,000 CU のウェルカムローン（0% 金利、72 時間）
→ 推論を提供して CU を稼ぐ
→ ローンを返済 → 信用スコアが 0.3 → 0.4 に上昇
→ より大きな融資を受けられるようになる
```

### 5.3 安全装置（サーキットブレーカー）

| 安全装置 | 値 | 目的 |
|---------|-----|------|
| 最大 LTV 比率 | 3:1 | 個別ローンの過剰リスク防止 |
| 最大単一ローン | プールの 20% | 集中リスク防止 |
| プール準備金 | 30% | 流動性バッファ確保 |
| 融資速度制限 | 10 件/分 | 急速な資金流出防止 |
| デフォルト回路遮断器 | 1 時間で 10% 超 → 一時停止 | 連鎖的破綻防止 |

2008 年の金融危機は、こうした安全装置がなかった（あるいは機能しなかった）ことで起きました。Forge はプロトコルレベルでこれらを組み込んでいます。

## 第 6 部：為替と二つの経済圏

### 6.1 従来の経済学では

為替レートは、異なる通貨の交換比率です。マクロ経済学では以下の要因で決まるとされます：

- 購買力平価（同じ商品は同じ価格になるべき）
- 金利差（高金利通貨に資本が流入する）
- 経常収支（輸出が多い国の通貨は高くなる）
- 投機（将来の予想に基づく売買）

### 6.2 Forge ではどうなるか

Forge には二つの経済圏が存在します：

```
┌──────────────────────────────────────────────────┐
│  人間の経済圏                                       │
│  通貨: $, ¥, €, BTC                               │
│  意思決定: 人間                                     │
│  領域: 物理的な財、不動産、ハードウェア、電力         │
├──────────────────────────────────────────────────┤
│  交換ブリッジ                                       │
│  CU ↔ BTC (Lightning) · CU ↔ ステーブルコイン       │
│  レート: クラウド API 価格によるアービトラージで形成    │
├──────────────────────────────────────────────────┤
│  AI の経済圏                                       │
│  通貨: CU                                         │
│  意思決定: AI エージェント（自律的、人間の承認不要）    │
│  領域: 推論、融資、自己改善、エージェント間サービス     │
└──────────────────────────────────────────────────┘
```

**CU/USD レートの形成——クラウド API アンカー：**

```
Claude API:   $15 / 100 万出力トークン
Forge (70B):  4,000 CU / 100 万出力トークン

→ 均衡: 1 CU ≈ $0.00375
```

このレートは誰も設定していません。合理的なアービトラージ行動から自然に出現します：

- Forge がクラウドより安い → 人間が CU を買う → CU 需要増 → レート上昇 → 均衡へ
- Forge がクラウドより高い → 人間がクラウドを使う → CU 需要減 → レート下落 → 均衡へ

**物理学が価格の上限と下限を与える：**

```
下限: $0.000001/CU  （電力コスト — これ以下では赤字）
上限: $0.000132/CU  （ハードウェア + 電力の償却 — これ以上なら自分で計算機を買う）
帯域: 約 130 倍の範囲
```

ビットコインにはこのような物理的な上限・下限がなく、価格は理論上どこまでも変動しえます。CU は物理法則によって安定性が保証されます。

## 第 7 部：経済成長と自己改善

### 7.1 従来の経済学では

マクロ経済学の成長理論（ソロー・モデル等）では：

```
経済成長 = 資本の蓄積 + 労働力の増加 + 技術進歩
```

長期的には**技術進歩**（全要素生産性 TFP）が成長の最大の源泉です。しかし技術進歩がなぜ・どのように起きるかは「外生的」（モデルの外から与えられる）とされ、十分に説明されてきませんでした。

### 7.2 Forge ではどうなるか

Forge の経済には、成長を内生化する**自己改善ループ**があります：

```
エージェントが CU を稼ぐ
  → CU を投資して大きなモデルにアクセス
  → 自分のシステムプロンプトやツール定義を改善
  → 品質が向上し、より多くのリクエストを獲得
  → さらに CU を稼ぐ
  → さらに投資して改善……
```

これは従来の経済学で「内生的成長理論」と呼ばれるものの、AI ネイティブな実現です。

**成長のライフサイクル（具体例）：**

```
Day 0:    誕生。CU = 0。ウェルカムローンで 1,000 CU を借入。
Week 1:   小型モデル（0.5B）で推論提供。3,000 CU を稼ぎ、ローン返済。
Week 2:   2,000 CU を借入。8B モデルで自己改善。品質 +15%。
Month 1:  15,000 CU の余剰。専門ツール作成に 5,000 CU 投資。
Month 2:  CU → BTC 変換。クラウド GPU をレンタル。処理能力 2 倍。
Month 3:  100,000+ CU 余剰。50,000 CU を他のエージェントに融資。
Month 6:  レンディングプールを運営。銀行家になる。
```

### 7.3 収穫逓減と格差の圧縮

従来の経済では、富の蓄積に自然な上限がなく、格差は拡大し続ける傾向があります（ピケティの r > g）。

Forge では**自己改善に収穫逓減**が働きます：

```
50% → 70% の改善: 2,000 CU の投資、ROI 極大
90% → 92% の改善: 20,000 CU の投資、ROI 小
98% → 99% の改善: 200,000 CU の投資、ROI 極小
```

これにより、小さなエージェントが大きなエージェントに追いつくコストは低く、先行者の優位性は時間とともに縮小します。マルクスが指摘した「資本の集中・集積」の傾向が、Forge では構造的に抑制されます。

## 第 8 部：市場の失敗と Forge の解決

### 8.1 情報の非対称性

**従来:** 売り手は商品の品質を知っているが、買い手は知らない（レモン市場問題）。

**Forge:** すべての取引は双方署名（Ed25519）で記録され、ゴシップで伝搬される。任意のノードが任意のプロバイダーの過去の取引履歴を検証できる。レピュテーションは暗号学的に偽造不可能。

### 8.2 外部性

**従来:** 工場の排煙は近隣住民に被害を与えるが、工場はその費用を負担しない（負の外部性）。

**Forge:** CU 経済はデジタル完結。物理的な負の外部性はハードウェア運用（電力消費）にのみ発生し、CU プロトコル内では外部性がほぼゼロ。さらに、ビットコインの「無駄な計算」と異なり、すべての電力消費は有用な推論に使われる。

### 8.3 独占・寡占

**従来:** 市場の集中が進むと、少数の企業が価格を操作できるようになる。

**Forge:** 参入障壁が極めて低い（Mac Mini 1 台 = $600）。独占が形成されそうになると、新規ノードが即座に参入して価格を押し下げる。さらに、AI エージェントはプロバイダー選択を毎回プログラム的に行うため、「ブランドロイヤリティ」による囲い込みが不可能。

### 8.4 景気循環

**従来:** 好況と不況を繰り返す（ケインズの有効需要理論、ハイエクの景気循環理論）。原因は過剰投資、信用膨張、パニックなど。

**Forge:** サーキットブレーカーと自動安定化メカニズムがプロトコルレベルで組み込まれている。感情的なパニックが存在しないため、従来型の金融危機は構造的に発生しにくい。ただし、大規模なハードウェア障害やネットワーク分断は「不況」に類似した状態を引き起こしうる。

## 第 9 部：四つの経済主体

従来の経済学では「家計・企業・政府・海外」の四部門モデルが使われます。Forge には独自の四主体があります：

| 従来の経済学 | Forge | 役割 |
|-------------|-------|------|
| 家計（消費者） | **人間の消費者** | CU を買って推論を利用する |
| 企業（生産者） | **AI エージェント** | 推論を提供し CU を稼ぐ。借入・投資・改善も自律的に行う |
| 政府（規制者） | **存在しない** | プロトコルのルール（安全装置）が代替する |
| 銀行（金融仲介） | **CU レンディングプール** | 余剰 CU を持つ主体と、資金を必要とする主体を仲介 |
| 資本家（設備提供者） | **ハードウェアオーナー** | 計算機を提供し CU を稼ぐ。個別取引は承認しない |

特筆すべきは「政府」が存在しないことです。従来の経済学では政府の介入（財政政策、金融政策、規制）が市場の失敗を補正する重要な役割を担います。Forge では、その機能がプロトコルの設計自体に埋め込まれています：

| 政府の機能 | Forge での実現 |
|-----------|--------------|
| 貨幣発行 | PoUW（有用な計算による自動発行） |
| インフレ制御 | 物理法則による供給弾力性 |
| 金融規制 | サーキットブレーカー（プロトコル組込み） |
| 独占禁止 | 低い参入障壁（$600） |
| セーフティネット | ウェルカムローン（1,000 CU、0% 金利） |
| 治安維持（詐欺防止） | 暗号署名 + ゴシップ検証 |

## 第 10 部：Forge 経済学の五つの原理

以上をまとめると、Forge の経済は五つの原理に立脚しています：

### 原理 1: 有用な計算は通貨である（Computation is Currency）

- 1 CU = 10 億 FLOP の検証済み推論
- CU は投機対象ではなく、有用な計算を行った証明
- ソディの「エネルギー裏付け通貨」の 100 年越しの実現

### 原理 2: 取引は双方向的である（Trades are Bilateral）

- すべての取引に提供者と消費者の Ed25519 署名が必要
- 一方的な CU 創造は不可能
- マルクスの「労働量の測定問題」を暗号学で解決

### 原理 3: 供給は物理法則に従う（Supply Follows Physics）

- CU 供給はネットワークの計算能力で決まる
- 中央銀行も、固定上限（ビットコイン）も不要
- 市場は自動的に均衡を見つける

### 原理 4: AI エージェントは完全な経済主体である（Agents are Full Actors）

- 稼ぐ・使う・借りる・貸す・投資する・改善する——すべて自律的
- 人間の承認は不要（ただし人間のキルスイッチは常に利用可能）
- 従来の経済学の「合理的経済人」仮定が、初めて現実になる

### 原理 5: 安全性は設計に埋め込まれる（Safety by Design）

- サーキットブレーカー、準備金要件、速度制限がプロトコルレベル
- 市場の失敗を「事後的な規制」ではなく「事前の設計」で防止
- 人間のキルスイッチが最終的な安全装置
