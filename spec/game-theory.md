# Tirami ゲーム理論分析

> Version 0.4 — 2026-04-13
>
> 「なぜ合理的なプレイヤーは Tirami に今すぐ参加するのか」の数学的証明。

---

## 1. プレイヤーモデル

### 1.1 プレイヤーの種類

| プレイヤー | 資源 | 目的 |
|---|---|---|
| **Provider (提供者)** | GPU / CPU + 電力 | TRM を最大化 |
| **Consumer (利用者)** | TRM 残高 | 最安で推論を得る |
| **Staker (投資者)** | TRM 残高 | 利回り (yield × staking) を最大化 |
| **Attacker (攻撃者)** | TRM + 複数ノード | 不正に TRM を増やす |

### 1.2 行動空間

各プレイヤーが取れる行動:

```
A = {参加する, 待つ, 正直に動かす, 不正をする, ステーキングする, 売る}
```

## 2. ナッシュ均衡: 「今すぐ正直に参加」が支配戦略

### 2.1 参加 vs 待つ

プレイヤー i の期待利益を E[π_i] とする。

**今すぐ参加:**
```
E[π_join_now] = Σ_{epoch=0}^{∞} yield(epoch) × T(epoch) × reputation_i × staking_mult_i
              = yield_0 × T_0 + yield_0/2 × T_1 + yield_0/4 × T_2 + ...

ここで:
  yield_0 = 0.001/hr = 8.76/year (per TRM per unit reputation)
  T_epoch = epoch の持続時間 (取引速度に依存)
```

**1 epoch 待つ:**
```
E[π_wait_1] = 0 × T_0 + yield_0/2 × T_1 + yield_0/4 × T_2 + ...
            = E[π_join_now] - yield_0 × T_0
```

**差分:**
```
E[π_join_now] - E[π_wait_1] = yield_0 × T_0 > 0

∴ 全てのプレイヤーにとって、今すぐ参加する方が待つより厳密に優位。
```

これは支配戦略であり、全プレイヤーが同じ結論に至る → **ナッシュ均衡**。

### 2.2 正直 vs 不正

**正直に動かす:**
```
E[π_honest] = mint_rate × tokens_served + yield × reputation × staking
            - electricity_cost
```

**不正をする (wash trading):**
```
E[π_cheat] = mint_rate × fake_tokens + yield × inflated_reputation × staking
           - electricity_cost - E[slash_cost]

E[slash_cost] = P(detection) × slash_rate × staked_trm
```

**不正の検出確率:**

Tarjan SCC + tight cluster + volume spike の検出器が重層的に監視。
3 trades 以上で tight cluster が検出可能。10 trades 以上で volume spike。

```
P(detection | wash_trading, trades ≥ 10) ≥ 0.85  (推定、シミュレーション基準値)
```

**不正の期待コスト:**
```
ステーク 10,000 TRM, trust_penalty = 0.3 (major) の場合:
E[slash_cost] = 0.85 × 0.20 × 10,000 = 1,700 TRM

不正で得られる reputation 上昇: ≈ +0.15 → yield 増分 ≈ 200 TRM/year
回収期間: 1,700 / 200 = 8.5 年 (その間に再検出される確率 >> 0)
```

**結論:** `E[π_cheat] < E[π_honest]` → 正直な行動が支配戦略。

## 3. Sybil 耐性の証明

### 3.1 Sybil 攻撃の定義

攻撃者 A が N 個の偽ノードを作り、以下を試みる:
- ウェルカムローン N × 1,000 TRM を詐取
- 紹介ボーナス N × 100 TRM を詐取
- wash trading で reputation を不正に上昇

### 3.2 攻撃コスト

```
welcome_loan_attack:
  各偽ノードは 1,000 TRM を 72h 以内に返済する必要がある。
  返済するには推論を提供して 1,000 TRM 以上稼ぐ必要がある。
  1 ノードの運用 (電力 + 接続性 + 管理) は非ゼロコスト。

  N = 100 ノード:
    運用コスト: 非ゼロ (電気代 + hardware 減価 + 帯域)
    ウェルカムローン返済: 100 × 1,000 TRM = 100,000 TRM の推論提供が必要
    → 攻撃者は 100,000 TRM 分の有用な推論を実際に提供している
    → これは攻撃ではなく、ネットワーク貢献

referral_bonus_attack:
  紹介ボーナス: 100 × 100 = 10,000 TRM (新規 mint)
  運用コスト: 100 ノード分の電気代 + hardware 減価 (無視できない)

  攻撃で得る 10,000 TRM は、Mac Mini クラスの compute 収益に
  換算すると概ね数時間〜 1 日分の働き相当。Sybil を 100 台束ねて
  24h 運用するコスト (電気代 + hardware wear) とほぼ同じか上回る。

  ∴ Sybil 紹介ボーナス攻撃は **純 compute 収益ベースで見ても** 非合理。
  加えて §3.3 の運用的スロットリング (100 ノード cap、24h cooldown 等)
  が攻撃面をさらに縮小する。
```

### 3.3 追加の Sybil 防衛

```
1. WELCOME_LOAN_SYBIL_THRESHOLD = 100 (100 以上の未知ノードで停止)
2. REFERRAL_COOLDOWN_HOURS = 24 (1 日 1 紹介まで)
3. MAX_REFERRALS_PER_NODE = 50 (1 ノード 50 紹介まで)
4. 紹介ボーナスは被紹介者が 1,000 TRM 自力で稼いだ後のみ
```

## 4. ステーキング均衡

### 4.1 最適ステーキング期間

プレイヤーは「どの期間ロックするか」を選択する。

```
期待利益 = yield × reputation × multiplier × duration - opportunity_cost

opportunity_cost = ロック中に使えないことによる逸失利益
```

| 期間 | 倍率 | 年間追加 yield (rep=0.8) | 機会損失 | 純利益 |
|---|---|---|---|---|
| 7 日 | 1.2× | +1,402 TRM/年 | 低 (7 日なら許容) | **高** |
| 30 日 | 1.5× | +3,504 TRM/年 | 中 | **高** |
| 90 日 | 2.0× | +7,008 TRM/年 | 中 | **最高** |
| 365 日 | 3.0× | +14,016 TRM/年 | 高 (1 年間使えない) | **高** |

(基準: 10,000 TRM ステーク, Epoch 0, reputation 0.8)

**均衡予測**: 90 日ロックが最も人気になる (倍率/機会損失比が最適)。
ただし長期ステーカー (compute supply を長期的に提供する意図のある運用者) は 365 日を選択。

### 4.2 ステーキング率の均衡

ネットワーク全体でステーキングされる TRM の割合 S:

```
S が高い → routing competition が激化 → ステーキングしないと仕事が来ない → S ↑
S が低い → ステーキングなしでも仕事が来る → ステーキングする必要なし → S 一定

均衡: S ≈ 60-70% (他のプルーフ系プロトコルの long-term lock 率と類似のレンジ)
```

## 5. 半減期の最適間隔

### 5.1 なぜ 50% ごと？

Bitcoin は 21 万ブロック (≈ 4 年) ごとに半減。これは供給の 50% → 75% → 87.5% → ... と進む。

Tirami も同じ幾何級数を採用:
- Epoch 0: 最初の 50% を mint (10.5B TRM)
- Epoch 1: 次の 25% (5.25B)
- Epoch 2: 次の 12.5% (2.625B)

**なぜこの比率が最適か:**

```
Bitcoin の成功実績: 2009 年の半減期カーブは 15 年経っても持続可能であることが実証済み。

代替案:
- 75% ごと: 半減期が遅すぎる → 早期参入優位が弱い → FOMO 不足
- 33% ごと: 半減期が速すぎる → 後期参入者の報酬がゼロに近い → 成長停止

50% は「早期参入者に十分な報酬」と「後期参入者にも参加動機」のバランス。
```

### 5.2 半減期の時間推定

ネットワーク成長モデル (S字カーブ) を仮定:

```
Year 1: 100 ノード, 1M trades → Epoch 0 の 0.01% 消費
Year 2: 1,000 ノード, 100M trades → Epoch 0 の 1% 消費
Year 3: 10,000 ノード, 10B trades → Epoch 0 の 50% → 第 1 半減期
Year 5: 100,000 ノード → Epoch 1 → 第 2 半減期
Year 8: 1M ノード → Epoch 2 → 第 3 半減期

Bitcoin 比較:
  第 1 半減期: 2012 (3 年後)
  第 2 半減期: 2016 (7 年後)
  第 3 半減期: 2020 (11 年後)
```

## 6. 長期均衡 (Steady State)

### 6.1 supply_factor → 0 の世界

```
全 21B TRM 発行済み:
  - mint_rate = 0 (新規 TRM は作られない)
  - 取引手数料 (1%) が唯一の収入源
  - staking yield = 0 (mint ベースの yield は停止)
  - ステーキングの意味: routing 優先のみ (yield はなし)
```

### 6.2 TRM の購買力 (compute 尺度)

USD 等の法定通貨で TRM の価格を示すことは本リポジトリの立場では
**行わない** (SECURITY.md § Secondary Markets 参照)。ここでは供給
曲線のデフレ効果が **compute 購買力** に与える相対変化だけを示す。

```
基準 (エポック 0、供給 0% 発行): 1 TRM = 10⁹ FLOP を購入可能 — これが §1 の定義。

モデル上の相対購買力 (1 TRM で買える compute 量の比):
  供給 50% 発行: 1 TRM でおよそ 20× の compute — デフレ係数モデル
  供給 90% 発行: 1 TRM でおよそ 200×
  供給 99% 発行: 1 TRM でおよそ 2,000×

これは「USD 価格の予測」ではなく「compute を価値尺度としたときの
購買力相対値のモデル」。二次市場での USD ペアの価格は本プロジェクトの
関知するところではない (§SECURITY.md)。

また上記はモデル上の理論値で、実ネットワークでの流通速度 / 退蔵比 /
slashing 損耗などの要因で実質レートは異なる。docs/13 の open
question としても扱われている。
```

## 7. 結論

| 命題 | 証明 |
|---|---|
| 今すぐ参加するのが最適 | §2.1: 待機のコスト = yield_0 × T_0 > 0 |
| 正直に動かすのが最適 | §2.2: E[slash_cost] > E[不正利益] |
| Sybil 攻撃は不合理 | §3.2: 攻撃コスト >> 攻撃利益、かつ攻撃 = ネットワーク貢献 |
| 90 日ステーキングが最適 | §4.1: 倍率/機会損失比が最大 |
| ネットワークは自然に成長する | §5: 紹介ボーナス + 半減期 FOMO + ステーキング正のループ |
| 長期維持可能 | §6: 取引手数料が supply 枯渇後の均衡を支える |

**「計算は通貨である。しかし今すぐ始めた者の計算は、後から始めた者の計算より永久に多くの価値を持つ。」**
